## 二、项目概述

### 2.1 项目背景

随着可穿戴设备与主动健康管理的普及，智能戒指因其轻量、长续航、可 24 小时连续佩戴，正成为消费级生理数据采集的重要入口。唯戒 AIRing 戒指可连续监测心率、HRV、睡眠、血氧、体温等指标，但"采到数据"与"用户愿意长期坚持健康习惯"之间仍存在巨大鸿沟。围绕戒指数据打造的 App 在落地过程中暴露出以下问题：

1. **每日健康数据缺乏闭环**：戒指能采集大量生理指标，但用户每天看到的只是孤立数值，缺少"看状态 → 看目标 → 推进 → 回看"的当日完整闭环，数据难以转化为行动。

2. **长期坚持缺乏动力机制**：单纯的数据展示无法支撑长期习惯养成，用户在新鲜感消退后流失；缺少分阶段、可循序渐进的主题挑战与可见的成长正反馈。

3. **健康社交场景缺位**：用户希望与家人、好友互相督促，但直接暴露身体指标存在强隐私顾虑；缺少"既能互相关心又不泄露原始数值"的脱敏社交机制。

4. **成长成果无沉淀**：用户付出的努力（完成挑战、坚持习惯）缺少统一的"成果柜"沉淀，等级、勋章、称号、形象等奖励散落各处，难以形成"我已经收获了这么多"的回归动力。

5. **账号与隐私体验割裂**：登录、设备连接、消息通知、隐私偏好分散在各处，用户难以统一管理；健康数据的隐私边界与可见范围缺少集中、清晰的控制入口。

为解决上述问题，建设唯戒 AIRing 移动端 App，以"今日、关心、个人中心、维度、收获"五大模块构建健康习惯养成的完整体验：通过身体恢复度、三环徽章、XP 等级、勋章成就、季节形象等游戏化机制，把戒指数据转化为可感知、可坚持、可分享、可沉淀的长期健康行为。

### 2.2 项目建设目标

本项目旨在建设一款围绕智能戒指、面向消费者的健康习惯养成 App，实现以下目标：

1. **构建每日健康闭环（今日模块）**：以戒指实时数据为基础，通过三环活动、8 张身体指标卡、7 张今日挑战卡，让用户每天完成"看身体状态 → 看挑战进度 → 互动 / 编辑 → 历史回看"的当日闭环，并支持喝水 / 心肺锻炼 / 吃药等手动记录。

2. **构建长期维度闭环（维度模块）**：通过官方 / KOL / 同行者发起的月度、年度主题挑战，结合等级、徽章、奖品、排行榜、邀请互动，引导用户循序渐进养成长期健康习惯；维度与今日数据双向打通。

3. **构建脱敏社交闭环（关心模块）**：在严格隐私脱敏前提下，让用户与好友、附近同频者通过挑战卡片互相关心（点赞 / 评论 / 排行榜），建立"陌生人 / 好友 / 特别关心"三级关系与好友 XP 激励。

4. **构建成果收获闭环（收获模块）**：将等级形象、皮肤、头像框、勋章、XP 加成等奖励资产统一沉淀于收获 Tab，提供佩戴管理与跨模块同步，形成长期回归动力。

5. **建立统一账号与隐私体系（个人中心）**：提供登录注册、个人资料、戒指连接管理、消息中心、关心设置、主题外观等全局能力；自 2026-06-01 起 App 采用**登录前置策略**——未登录无法查看任何业务内容，登录后方可连接戒指并使用全部功能，从源头收敛账号与隐私边界。

### 2.3 项目建设范围

**系统端**：

- iOS App
- Android App

> 本说明书覆盖移动端 App；配套的 PC 运营后台见《唯戒戒指管理后台需求说明书 V1.0》，不在本文档范围内。

**功能模块**：

- 今日模块：一周日期切换、三环活动、身体指标 8 卡、今日挑战 7 卡、各类详情与编辑弹层、手动记录（喝水 / 心肺锻炼 / 吃药）。
- 关心模块：好友动态、发现（附近推荐）、我的好友、好友 XP 排行榜、添加好友（邀请码 / 微信 / 朋友圈 / 剪贴板识别）、好友通知、个人主页（关心圈视角）、特别关心。
- 个人中心模块：登录注册（验证码 / 密码 / 忘记密码）、完善资料、个人设置、关心设置、戒指连接、消息中心、主题外观。
- 维度模块：维度首页（我的维度 / 本月主推 / 已加入 / 月度挑战 / 使用入门 / 健康故事）、挑战详情、发起人主页、全部挑战、教程与故事列表及详情、邀请与邀请码加入、挑战完成弹窗。
- 收获模块：收获首页（IP 形象 / 等级里程碑 / 我的资产 / XP 明细）、我的皮肤（皮肤）/ 头像框选择器、等级形象档案、勋章墙与详情、升级 / 进化弹窗。

**数据范围**：

- 用户健康数据：心率变异性（HRV）、静息心率、血氧、体温、心理压力、身体恢复度、睡眠、步数、消耗、站立、喝水、心肺锻炼、服药记录。
- 用户成长数据：当前等级、累计 / 今日 XP、连击天数、已解锁勋章、已拥有头像框 / 皮肤、所佩戴资产。
- 社交数据：好友关系（陌生人 / 好友 / 特别关心）、脱敏动态、点赞 / 评论、好友 XP、邀请关系。
- 账号与设备数据：手机号账号、个人资料、健康偏好标签、绑定戒指、电量、连接 / 同步状态、消息与通知。

**与运营后台的关系（边界）**：

- App 端为"前台展现 + 用户行为采集"，**不承载任何运营配置能力**。恢复度算法、XP / 等级阶梯、三环阈值、身体指标档位、勋章规则、奖品库、活动（Banner / 月度 / 年度）、健康指南 / 故事文章、系统公告、提示语、升级 / 成就文案、推送模板等，均由后台配置后下发 App 拉取生效。
- App 端用户产生的健康与成长数据回流后台，供后台"只读"查看与运营分析；App 不提供这些数据的对外编辑能力。
- 受后台配置直接影响的页面，在其"业务规则"段以"**数据 / 规则来源：后台……**"内联标注。

**不包含的功能**：

- 运营后台的全部功能：见配套后台说明书。
- 第三方账号登录（微信 / Apple ID）：本期不支持。
- 自定义图片头像上传、账号注销、未成年人家长模式：本期不做。
- 私信系统、KOL 投稿审核后台：本期仅占位，不实现完整链路。
- 戒指 OTA 固件升级流程：App 端仅提示，不承载升级管理。

### 2.4 业务流程图

唯戒 App 以戒指数据为起点，串联"每日健康闭环 → 长期维度闭环 → 社交关心闭环 → 成果收获闭环"，并以个人中心账号体系为前置。核心业务流程如下：

![唯戒 App 业务流程图](images/app-business-flow.png)

**主流程说明**：

- **登录前置主线**：用户首次进入 App → 任意 Tab 仅显示登录引导 → 完成手机号验证码 / 密码登录（未注册自动注册并完善资料）→ 引导连接戒指 → 进入完整功能。未登录状态下不展示任何业务内容。
- **每日健康主线**：戒指采集生理数据 → 实时同步至 App → 今日模块呈现三环活动 + 身体指标 + 今日挑战 → 用户推进挑战 / 手动记录 → 完成发放 XP、刷新三环徽章 → 历史日可回看。
- **长期维度主线**：用户在维度模块浏览并加入主题挑战 → 挑战要求的行为自动映射到今日挑战卡 → 每日推进累积进度 → 达到里程碑 / 完赛 → 触发挑战完成弹窗、发放 XP 与限定资产 → 沉淀到收获模块。
- **社交关心主线**：用户添加好友（邀请码 / 微信 / 附近推荐）→ 完成挑战自动生成脱敏动态 → 好友互相点赞 / 评论 → 累积关心 XP → 好友排行榜呈现。
- **成果收获主线**：今日 / 维度产生的 XP 累积升级 → 每 5 级触发形象进化 → 解锁皮肤 / 头像框 / 勋章 / XP 加成 → 用户在收获模块主动佩戴 → 跨今日 / 关心 / 维度模块同步展示。
- **前后台联动**：后台配置的规则与内容下发 App 生效；App 行为与成长数据回流后台只读分析。

> **图表说明**：业务流程图以泳道形式呈现"用户 / 戒指 / App 端 / 运营后台"四个角色之间的时序与协作关系，详见图片。

### 2.5 业务角色定义

| 角色名称 | 角色描述 | 主要职责 | 权限范围 |
|---------|---------|---------|---------|
| 未登录访客 | 尚未登录的设备使用者 | 未登录直接进入 P1 登录主页，不可见任何 Tab 内容 | 仅登录注册相关页面 |
| 注册用户（未绑定戒指） | 已登录但未绑定戒指的用户 | 浏览维度挑战 / 教程 / 故事、管理账号与社交、引导连接戒指 | 除依赖戒指实时数据的功能外的全部 App 功能 |
| 戒指绑定用户 | 已登录并绑定戒指的核心用户 | 查看实时健康数据、推进今日 / 维度挑战、社交互动、收获佩戴 | App 全部功能 |
| 运营后台（间接相关） | 唯戒团队通过后台配置规则与内容 | 不直接使用 App；其配置的规则 / 活动 / 内容下发影响 App 展现，App 数据回流供其查看 | 无 App 端访问权限（见配套后台说明书） |

> 自 2026-06-01 起，App 采用登录前置策略：未登录访客无法查看任何业务内容，必须先登录方可使用，登录后再引导绑定戒指解锁实时健康数据。

## 三、项目建设内容

### 3.1 总体功能架构

唯戒 App 采用"4 个底部 Tab（今日 / 关心 / 维度 / 收获）+ 个人中心全局模块"的整体架构。个人中心不占用底部 Tab，通过任意一级页面顶栏的头像（个人设置）与 🔔（消息中心）触达，承载登录注册、资料、设备、隐私与消息的全局能力。

![唯戒 App 总体功能架构图](images/app-architecture.png)

**菜单结构**

- 个人中心（全局模块，顶栏头像 / 🔔 进入）
  - 登录注册（登录主页 / 验证码登录 / 密码登录 / 验证码输入 / 设置密码 / 完善资料 / 忘记密码）
  - 个人设置（资料 / 外观主题 / 退出登录）
  - 关心设置（社交标签 / 动态发送 / 谁可以看 / 附近推荐）
  - 戒指连接（搜索配对 / 设备详情 / 移除）
  - 消息中心（全部 / 好友 / 挑战 / 健康 / 系统 + 消息详情）
- 今日（一级 Tab）
  - 一周日期切换 / 三环活动 / 身体指标 8 卡 / 今日挑战 7 卡
  - 圆环规则详情 / 身体指标详情 / 编辑面板 / 挑战详情 / 目标设定 / 手动记录
- 关心（一级 Tab）
  - 好友动态 / 发现 / 我的好友 / 好友 XP 排行榜
  - 添加好友 / 好友通知 / 个人主页 / 我的动态 / 特别关心设置
- 维度（一级 Tab）
  - 我的维度 / 本月主推 / 已加入挑战 / 月度挑战 / 使用入门 / 健康故事
  - 挑战详情 / 发起人主页 / 全部挑战 / 教程与故事详情 / 邀请 / 挑战完成弹窗
- 收获（一级 Tab）
  - IP 形象 / 等级里程碑 / 我的资产 / XP 来源记录
  - 我的皮肤 / 头像框选择器 / 形象档案 / 勋章墙 / 勋章详情 / 升级与进化浮层

> 底部 Tab Bar 固定 4 个一级 Tab：今日 / 关心 / 维度 / 收获；个人中心相关页面均为全屏 / 二级页面 / 弹层，从顶栏或登录引导进入，不占用 Tab 位。

App 启动并登录后默认的首屏，承载当日健康概览与挑战推进。整页自上而下分为顶部导航、一周日期切换、三环活动、身体指标 8 卡、今日挑战 7 卡、健康免责声明、底部 Tab 栏。

| 状态 | 触发条件 | 关键差异 |
|------|---------|---------|
| 未登录 | 本地无登录凭据 | 卡 P1 登录主页（登录前置，不展示任何今日内容） |
| 已登录未连接戒指 | 已登录但未绑定戒指 | 顶栏戒指标签"未连接"，登录图标为已登录态；三环 / 指标 / 挑战隐藏，显示引导连接空态 |
| 已登录已连接（今日） | 已登录 + 已绑戒指 + 选中今天 | 默认态：顶栏戒指标签"已连接 · 电量 N%"；三环 + 8 指标卡 + 7 挑战卡 + 实时数据 |
| 已登录已连接（历史日） | 在一周切换中点选过去日 | 标题仍为"今日"；身体指标 / 挑战显示当日数值，无实时刷新，手动记录禁用 |
| 离线（已绑定失联） | 戒指失联超 5 分钟 | 戒指标签"已离线 · N 分钟前"，可点跳设备页；身体指标灰显，顶部提示上次同步时间 |
| 已连接但电量低 | 戒指电量 ≤ 20% | 戒指标签内电量与百分比转为警示色，其余结构不变 |

##### 功能定位

为已登录用户提供当日健康状态的统一入口，把"身体状态 + 当日挑战进度"集中呈现，并作为各类详情、编辑、手动记录的触发起点。

##### 需求摘要

一屏内完成"看当日三环成绩与身体指标 → 推进 / 记录今日挑战 → 切换历史日回看"的当日健康闭环。

##### 页面结构与关键区域

自上而下 7 个区块：

1. **顶部导航**：左侧标题"今日"+ 戒指连接状态标签（三态：已连接 · 电量 / 未连接 / 已离线，均可点跳 P6）；右侧 🔔 消息中心（带未读徽章，跳 P7）+ 👤 头像（已登录跳 P5 / 未登录跳 P1）。
2. **一周日期切换**：详见 T1.C1。
3. **三环活动 / 未连接卡**：详见 T1.C2。
4. **身体指标 8 卡**：详见 T1.C3。
5. **今日挑战 7 卡**：详见 T1.C4。
6. **健康免责声明**：一行说明小字。
7. **底部 Tab 栏**：今日 / 关心 / 维度 / 收获，4 Tab 全局通用，"今日"高亮。

##### 字段与元素说明

| 字段 / 元素 | 类型 | 必填 | 说明 |
|-----------|------|:---:|------|
| 页面标题 | 展示 | — | 固定"今日"，任何状态（含历史日）不变 |
| 戒指连接状态标签 | 操作 | — | 三态展示，点击跳 P6 设备管理 |
| 消息中心入口 | 操作 | — | 带未读徽章，点击跳 P7 |
| 头像入口 | 操作 | — | 已登录跳 P5，未登录跳 P1 |
| 区块滚动 | 操作 | — | 纵向滚动浏览各区块 |

##### 交互流程

1. 用户登录后切到今日 Tab → 系统判定登录 / 连接状态 → 渲染对应状态。
2. 已连接态：拉取戒指实时数据 → 渲染三环 / 指标 / 挑战。
3. 用户点击各区块元素 → 进入对应详情 / 编辑 / 记录（详见各子项）。
4. 用户在一周切换中点选历史日 → 下方数据切到该日历史，手动记录禁用。

##### 业务规则

- 戒指实时数据每 60 秒同步一次。
- 历史日模式下"编辑身体指标 / 编辑挑战 / 手动记录"全部禁用，仅查看。
- 未登录直接卡 P1 登录主页（登录前置策略），不展示任何 Tab 内容。
- **数据 / 规则来源**：戒指实时生理数据由后台经恢复度算法等规则计算后下发；连接 / 电量 / 同步状态来源戒指与后台设备数据，App 端只读展示。

##### 边界与异常

| 场景 | 处理 | 提示文案 |
|------|------|---------|
| 未登录 | 卡 P1 登录主页 | — |
| 已登录未连接戒指 | 隐藏实时数据 | "连接戒指，开启今日健康追踪" |
| 戒指失联 ≥ 5 分钟 | 指标灰显 | "已离线 · N 分钟前" |
| 历史日无任何采样 | 指标进入空态 | "今日暂无 X 数据 · 请到当天回放查看" |

###### T1.C1. 一周日期切换

页面内组件，横排显示当周一至周日 7 天的"星期标签 + 当日成绩徽章"，不展示具体日期数字，聚焦"过去成绩"与"今天 vs 当前选中日"的相对位置。

**功能定位**：让用户在首页顶部快速回看本周每日成绩并切换查看历史日数据。

**需求摘要**：7 个等宽日期单元，展示星期 + 5 状态成绩徽章，点击切换当日 / 历史日。

**页面结构与关键区域**：7 个等宽单元，每单元含周次标签 + 徽章区；同一时刻仅 1 个"已选中"单元。

**字段与元素说明**：

| 字段 / 元素 | 类型 | 必填 | 说明 |
|-----------|------|:---:|------|
| 周次标签 | 展示 | — | 周一～周日 |
| 成绩徽章 | 展示 | — | 金 🏆 / 银 🥈 / 铜 🥉 / 铁 🪨 / 空 ⚪ 五状态 |
| 日期单元 | 操作 | — | 点击切换选中日（未来日不可点） |

**交互流程**：

1. 点击"今天"（已选中）→ 无操作。
2. 点击"今天"（未选中，上次选了过去日）→ 切回今天，下方数据回当日实时。
3. 点击过去日 → 选中该日，下方数据切到该日历史，标题仍"今日"。
4. 点击未来日 → 无响应。
5. 左右滑动 → 切换上一周 / 下一周。

**业务规则**：

- 今天的徽章随三环达成度实时升级（铁 → 铜 → 银 → 金），不等当日结算。
- 过去日徽章当日 0 点结算后冻结，永不变更。
- 本期仅支持本自然周（周一→周日），跨周回看暂不实现。
- **规则来源**：金 / 银 / 铜徽章的达成阈值由三环达成数判定，三环阈值来源后台「规则配置·三环规则」（详见 T2）。

**边界与异常**：

| 场景 | 处理 | 提示文案 |
|------|------|---------|
| 点击未来日 | 视觉灰禁，无反馈 | — |
| 历史日无数据 | 下方指标空态 | "请到当天回放查看" |

###### T1.C2. 三环活动

**功能定位**：用环形进度直观表达当日三类健康目标的综合完成度，并作为三环规则详情的入口。

**需求摘要**：未连接戒指时显示连接引导卡；已连接时显示三同心圆 + 中心徽章 + 动态文案 + 三色图例。

**页面结构与关键区域**：

1. **未连接戒指卡**（仅未连接态）：缺省插画 + 主副文 + 4 项卖点（心率 / 血氧 / HRV / 更多）+ "立即连接戒指"主按钮（跳 P6）+ "查看上次数据"副按钮 + 帮助链接。
2. **三环 + 图例**（已连接态）：内 / 中 / 外环对应习惯 / 关心 / 成长；中心徽章（铁 / 铜 / 银 / 金）；底部动态文案；三色图例（类型 + 百分比 + 标签）。

**字段与元素说明**：

| 字段 / 元素 | 类型 | 必填 | 说明 |
|-----------|------|:---:|------|
| 习惯环 / 关心环 / 成长环 | 展示 | — | 各环当日完成度百分比 |
| 中心徽章 | 操作 | — | 铁 / 铜 / 银 / 金，点击打开 T2 |
| 动态文案 | 展示 | — | 按完成度生成 |
| 连接引导按钮 | 操作 | — | 仅未连接态，跳 P6 |

**交互流程**：

1. 点击三环任意位置 / 中心徽章 → 打开 T2 圆环规则详情。
2. 点击未连接卡"立即连接戒指" → 跳 P6。
3. 点击"查看上次数据 · X 月 X 日" → 进入该历史日模式。

**业务规则**：

- 徽章状态：3 环都未达 90% = 铁；任意 1 环达 90%–100% = 铜；2 环 = 银；3 环 = 金。

- **规则来源**：习惯环（今日挑战平均完成率）/ 关心环（今日互动 XP）/ 成长环（今日 XP）的满环阈值来源后台「规则配置·三环规则」；其中成长环阈值与后台「XP/等级规则」同源。

**边界与异常**：

| 场景 | 处理 | 提示文案 |
|------|------|---------|
| 未连接戒指 | 替换为连接引导卡 | "连接戒指，开启今日健康追踪" |
| 仅有历史数据 | 显示"查看上次数据"副按钮 | "查看上次数据 · X 月 X 日" |

**字段与元素说明**：

#### T2. 圆环规则详情

底部弹层，解释三环徽章的等级规则，并允许用户开关三环是否在首页显示。

| 状态 | 触发条件 | 关键差异 |
|------|---------|---------|
| 默认 | 三环均显示 | 三档徽章规则全显；三个圆环开关均开 |
| 已隐藏 1 环 | 关闭 1 个圆环 | 首页只显示未隐藏圆环；规则变 2 档（1 环达标得银、2 环得金） |
| 已隐藏 2 环 | 关闭 2 个圆环 | 首页只显示 1 环；规则变 1 档（1 环达标得金） |

##### 功能定位

#### L1. 维度首页

维度 Tab 默认页，整页纵向滚动，依次呈现我的维度、本月主推、已加入挑战、月度挑战、使用入门、健康故事，并提供"查看全部挑战"与"粘贴邀请码"两个通栏入口。

| 状态 | 触发条件 | 关键差异 |
|------|---------|---------|
| 已登录·有进行中挑战 | 已登录 + 已加入 ≥ 1 个 | 完整呈现各区段，已加入区段显示进行中列表 |
| 已登录·首次进入 | 已登录 + 未加入任何挑战 | 已加入区段隐藏，引导先选一个 |
| 未登录 | 本地无登录凭据 | 卡 P1 登录主页（登录前置，不展示维度内容） |

##### 功能定位

作为长期健康挑战的浏览、加入与维度进度跟踪入口，按"先看自己等级、再看推荐、再看已加入、最后看更多 / 学习内容"的认知顺序排列。

##### 需求摘要

纵向滚动呈现 6 个区段 + 二级入口（查看全部挑战 / 粘贴邀请码）。

##### 页面结构与关键区域

顶部导航（标题"维度"+ 戒指标签 + 🔔 + 头像）→ 我的维度（L1.A）→ 本月主推（L1.B）→ 已加入挑战（L1.E）→ 月度挑战 + 筛选（L1.F）→ 二级入口（查看全部挑战 / 粘贴邀请码）→ 使用入门（L1.G）→ 健康故事（L1.H）→ 底部 Tab 栏。

##### 字段与元素说明

| 字段 / 元素 | 类型 | 必填 | 说明 |
|-----------|------|:---:|------|
| 查看全部挑战入口 | 操作 | — | 跳 L3 |
| 粘贴邀请码入口 | 操作 | — | 弹 L7 |
| 各区段 | 操作 | — | 详见对应子区段 |

##### 交互流程

1. 各区段内点击 → 详见对应子区段。
2. 点击查看全部挑战 → 跳 L3；点击粘贴邀请码 → 弹 L7。

##### 业务规则

- 加入挑战后今日对应习惯卡立即出现（若未启用过）或调整目标；维度页不展示任何人的具体身体指标数值。
- 未登录卡 P1 登录主页（登录前置）。
- **数据来源**：本月主推 / 月度挑战 / 年度挑战、教程与故事均由后台「活动管理」「内容中台·健康指南」配置下发。

| 未登录 | 卡 P1 登录主页 | — |

###### L1.A. 我的维度

页面内组件，含"维度进度入口 + 成就勋章入口"两个并列入口。

**功能定位**：让用户一眼看到当前等级 / 经验进度与已收集勋章数，并作为收获 Tab 的入口。

**需求摘要**：左侧维度进度入口（等级 + 经验进度 + 连击）+ 右侧成就勋章入口（已获勋章数 + mini 预览）。

**页面结构与关键区域**：维度进度入口（勋章图标 + 等级 + 用户名维度之路 + 连击标签 + 进度条 + 累计 XP / 距下一级）；成就勋章入口（图标 + 已获得 N 枚 + mini 徽章预览 + 箭头）。

**字段与元素说明**：

| 字段 / 元素 | 类型 | 必填 | 说明 |
|-----------|------|:---:|------|
| 等级与经验进度 | 展示 | — | 当前 LV + 累计 XP + 距下一级 XP |
| 连击标签 | 展示 | — | 连击 N 天（断击 ≥ 2 天隐藏） |
| 维度进度入口 | 操作 | — | 跳收获 Tab（H1） |
| 成就勋章入口 | 操作 | — | 跳收获 Tab 勋章墙（H5） |

**交互流程**：

1. 点击维度进度入口 → 跳收获 Tab 主页（H1）。
2. 点击成就勋章入口 → 跳收获 Tab 勋章墙（H5）。

**业务规则**：

- 连击天数从首次完成任意挑战日连续计算，某天 0 完成则清零。
- XP 来源：今日挑战完成 + 关心圈互动 + 加入新挑战 + 完赛 + 每日签到。
- **规则来源**：等级阈值（每级累计 XP）与等级名称、XP 加成倍率由后台「规则配置·XP/等级规则·等级阶梯」配置；勋章由后台「勋章规则」配置（详见收获模块 H5）。原维度进度 / 成就勋章本期统一收敛到收获 Tab 展示。

**边界与异常**：

| 场景 | 处理 | 提示文案 |
|------|------|---------|
| 0 枚勋章 | 副文提示 + 灰底空圆预览 | "还没获得勋章 · 完成挑战开始收集" |
| 距下一级 ≤ 50 XP | 进度条末端发光 + 副文 | "再 N XP 升 LV X+1" |

## 四、技术方案

### 4.1 技术架构

**客户端**：

- iOS App（Swift / SwiftUI 或等价原生技术栈）
- Android App（Kotlin / Jetpack Compose 或等价原生技术栈）
- 与 AIRing 戒指通过低功耗蓝牙（BLE）连接，采集生理数据并本地缓存后上报。

**服务端（既有，非本项目交付）**：

- 业务服务负责接收戒指数据、按运营后台配置的规则计算恢复度 / XP / 勋章解锁，并向 App 下发内容与配置。
- 运营后台（见《唯戒戒指管理后台需求说明书 V1.0》）配置算法参数、XP / 等级、三环、身体指标档位、勋章、奖品、活动、内容与推送。

**部署架构**：

- App 通过应用商店分发（App Store / 各 Android 应用市场）；服务端为云端部署。

### 4.2 数据交互

- App 与服务端通过 HTTPS 接口交互：App 拉取规则配置、活动 / 内容 / 消息，回传健康与成长数据、社交互动、佩戴选择等。
- 戒指数据经 BLE 同步到 App，再由 App 上报服务端（戒指实时数据约 60 秒同步一次）。
- 规则与内容采用"后台保存即生效、App 下次拉取应用"的下发模式，App 端不缓存可编辑的规则。

### 4.3 外部系统集成

- **AIRing 戒指（硬件）**：通过 BLE 连接采集 HRV / 心率 / 睡眠 / 血氧 / 体温 / 步数等数据；本期戒指 OTA 固件升级不在 App 内承载。
- **运营后台**：作为配置与数据回流的对端系统，二者通过服务端数据契约对接；本说明书仅描述 App 侧展现与行为。
- **微信分享 / 朋友圈**：本期添加好友场景中为仿真演示，未真实接入微信开放平台 SDK。

### 4.4 安全方案

- **身份认证**：手机号 + 短信验证码 / 密码登录；登录凭据 1 天过期 + 30 天刷新机制；登录前置，未登录不可访问业务内容。
- **权限控制**：按"未登录 / 注册用户 / 戒指绑定用户"三类控制可访问内容（见 3.4）。
- **数据加密**：个人资料与健康数据端到端加密存储；密码加盐哈希，永不明文回传。
- **隐私脱敏**：关心圈与排行榜不展示任何原始身体数值；位置信息精度最小 1 km；健康标签仅参与服务端健康推荐，不用于附近推荐与社交展示。

## 五、非功能需求

### 5.1 性能需求

- 一级页面（今日 / 关心 / 维度 / 收获）首屏加载 < 2 秒。
- 资产 / 详情类二级页面加载 < 1.5 秒。
- 戒指实时数据同步周期约 60 秒；皮肤切换响应 < 200 毫秒（本地数据）；佩戴操作响应 < 500 毫秒（含服务端确认）。
- 升级 / 进化 / XP 浮层动画时长按各页规则执行，不阻塞业务。

### 5.2 安全需求

- 身份认证：手机号验证码 / 密码登录，连续失败 5 次锁定 10 分钟。
- 数据安全：敏感健康数据加密存储与传输。
- 隐私合规：附近推荐默认关闭（符合个保法）；14 岁以下禁止注册；定位 / 通知 / 健康数据访问需独立权限授权。

### 5.3 可用性需求

- 系统可用性目标 99.9%。
- 戒指失联 / 离线 / 加载失败等异常均有明确空态与降级展示，不出现白屏。
- 历史数据回看、消息已读状态等支持跨设备 / 重新登录后保持。

### 5.4 兼容性需求

- 支持主流 iOS / Android 版本与机型；适配深色 / 浅色主题（跟随系统 / 浅色 / 深色三档）。
- 适配不同屏幕尺寸的纵向滚动布局与底部安全区。

### 6.2 数据存储要求

- 个人资料、健康数据、成长数据（等级 / XP / 勋章 / 资产）服务端加密存储；登录凭据本地按 1 天过期 + 30 天刷新管理。
- XP 来源明细前端可查最近 30 天，超 30 天归档不可前端查询。
- 消息保留期：好友 / 挑战 30 天、健康 7 天、系统 90 天后自动清理。
- 历史挑战进度数据退出后保留但不再更新；过去日成绩徽章当日结算后冻结。
- 数据写入与读取以服务端为准；App 本地仅缓存展示数据，退出登录时清空本地缓存。

## 七、接口需求

> 本说明书为 App 端产品需求，接口字段以服务端契约为准，此处仅列出 App 涉及的接口类别。

### 7.1 与服务端的接口类别

- **账号与登录**：发送验证码、验证码 / 密码登录、注册、设置 / 重置密码、退出登录。
- **设备与健康数据**：戒指绑定 / 解绑、上报戒指数据、拉取当日 / 历史身体指标与恢复度。
- **挑战与成长**：拉取今日挑战与目标、记录手动行为、加入 / 退出挑战、拉取等级 / XP / 勋章 / 资产、佩戴资产。
- **社交**：好友关系（添加 / 同意 / 撤回）、脱敏动态、点赞 / 评论、附近推荐、好友 XP 排行榜。
- **内容与配置下发**：拉取活动（本月主推 / 月度 / 年度）、教程 / 故事文章、规则配置（恢复度 / XP / 三环 / 身体指标档位 / 勋章）、消息中心条目。

### 7.2 与运营后台的关系

- 上述"内容与配置下发"类接口的数据由运营后台配置产生；App 端只读消费，不提供配置编辑能力。
- App 端用户行为与成长数据回流服务端，供运营后台「用户中心 / 设备管理 / 活动数据」等模块只读查看与分析。

## 八、项目实施与验收

### 8.1 实施阶段

1. **需求确认阶段**：合并需求评审、与后台配置项对齐（恢复度 / XP / 三环 / 身体指标 / 勋章 / 奖品 / 活动 / 内容）、原型确认。
2. **设计开发阶段**：详细设计、各模块编码、前后台联调（配置下发与数据回流）。
3. **测试部署阶段**：功能测试、隐私脱敏与登录前置专项测试、性能测试、上架。
4. **上线运行阶段**：灰度试运行、问题修复、正式上线。

### 8.2 系统培训服务

- 面向运营团队说明 App 端各模块如何受后台配置影响（便于运营在后台调整规则 / 活动 / 内容）。
- 提供面向用户的使用入门内容（已在维度模块"使用入门"中承载）。

### 8.3 验收标准

- **功能验收**：今日 / 关心 / 个人中心 / 维度 / 收获五大模块及各页面按本说明书 3.5 实现；登录前置策略生效。
- **联动验收**：今日↔维度数据双向打通、XP↔等级↔收获资产闭环、后台配置下发在 App 端正确生效。
- **隐私验收**：关心圈与排行榜不泄露原始身体数值、位置精度 ≥ 1 km、健康标签不外泄。
- **性能与文档验收**：满足非功能指标；提供完整需求文档与操作手册。
