## 1. 设计原则

1. **一切走 Token**：颜色、字号、间距、圆角、阴影一律使用 CSS 变量，不允许在业务页面里写死 hex/像素。
2. **同源同色**：同一类信息在不同页面必须使用同一个语义 Token（如"已完成"恒为 `--color-semantic-success`）。
3. **iOS 体感优先**：弹窗、按钮顺序、滚动手势遵循 iOS 习惯（左取消 / 右确认）。
4. **隐私默认收紧**：涉及身体数据、药品、位置信息默认不外传，靠用户主动开启。
5. **可扩展**：组件可通过修饰符（`.is-success` / `.btn-primary` 等）变体，但禁止 fork 出"风格相近的新组件"。

## 5. 图标系统

- 基于 `antd-mobile-icons`，统一使用 `<svg class="icon"><use href="#icon-xxx"/></svg>` 引用 SVG Sprite。
- 尺寸类：`.icon-sm`(16) / `.icon-md`(20) / `.icon-lg`(24) / `.icon-xl`(28) / `.icon-2xl`(32)。
- 颜色随父级 `color` 走（path 内置 `fill="currentColor"`）。
- 健康专用图标在 antd-mobile 中没有，使用 emoji（💜 ❤️ 🌙 🫁 👟 💧 🔥 🧍 🏃 💊）作为近似替代。
- Tab 栏使用 outline + fill 一对（已选 fill / 未选 outline）。

### 6.1 顶部导航

App 顶部由两层叠加：
- **第 1 层**：`.ios-status-bar` 系统状态栏（高度 44px，承载时间 / 信号 / 网络 / 电量）
- **第 2 层**：`.app-topbar--primary` 应用顶栏（高度 52px，承载页面名称、戒指连接状态、主题切换、个人设置）

两层共用同一磨砂背景，从屏顶 0 到 96px 形成连续的"磨砂帽"。

#### 系统状态栏 `.ios-status-bar`（独立组件）

iOS 风格的系统状态栏，复刻原生 iPhone（iOS 16+）形态。**全 App 通用**，所有需要显示状态栏的页面（一级 / 二级 / 全屏 / 弹层调起前）必须复用本组件，禁止单独画。

##### 结构

```
[ 时间 ]                                  [ 信号 4 bars ]  [ 网络文字 ]  [ 电池胶囊 ]
←─ 左侧 ─                                  ←──────── 右侧分组（间距 5px）────────→
```

| 元素     | 类名                       | 视觉与规则                                                                                                  |
| ------ | ------------------------ | ------------------------------------------------------------------------------------------------------ |
| 时间     | `.iss-time`              | 17/600，等宽数字（`font-feature-settings: "tnum"`），字距 -0.4px。文案动态（系统时间，原型固定 9:41）。                          |
| 信号     | `.iss-signal` + 4 ×`.iss-bar` | 4 根竖条，宽 3px，从左到右高度递增 3 / 5 / 7 / 9px，间距 2px；满信号时全黑，弱信号时尾几根降为 `--color-text-tertiary`。                  |
| 网络     | `.iss-network`           | 15/600，字距 -0.4px。文案 `5G` / `4G` / `LTE` / `Wi-Fi` 等。                                                    |
| 电池胶囊   | `.iss-battery`           | 黑底白字胶囊（背景 `--color-text-primary`，文字 `--color-text-on-dark`），宽 26px、高 13px、圆角 3.5px；右侧通过 `::after` 画 1.5×5px 的正极小凸起。 |
| 电池百分比  | `.iss-battery-pct`       | 10/700 + 等宽数字，居中。文案 = 整数百分比（不带 `%`，如 `83`）。                                                            |

##### 容器与色调

- 高度固定 44px，左右内边距 22px。
- 横向 `flex` 布局，时间靠左、`.iss-right` 容器靠右。
- 颜色全部继承自 `currentColor`（默认 `--color-text-primary`），便于深色模式翻转为白色。
- 背景**默认透明**（由父级提供），如需贴在毛玻璃面板上请由父级负责 `backdrop-filter`；本原型中嵌入 `.app-topbar--primary` 上方时与之共用一片白色 70% 毛玻璃。
- 中央 notch / 灵动岛区域视觉穿出由父级 `.notch` / `.dynamic-island` 元素覆盖。

##### 状态变体

| 修饰符                  | 用途                                                       |
| -------------------- | -------------------------------------------------------- |
| `.is-on-glass`       | 添加白色 70% 毛玻璃底；用于贴在透明 / 浅色页面上方。                           |
| `.is-on-dark`        | 文字反白（`--color-text-on-dark`），电池胶囊底变白、字变黑；用于深色模式或暗底页面。    |
| `.is-low-battery`    | 电池胶囊底色变 `--color-semantic-danger`，预警系统电量过低（≤ 20%）。       |

#### 一级页面 `.app-topbar--primary`

四个一级页面（**今日 / 关心 / 练级 / 收获**）的顶部标题栏结构完全统一，共四个元素：

```
[ 标题 ]  [ 戒指连接状态 ]               [ 🌙 主题切换 ]  [ 👤 个人设置 ]
←──── 左侧分组 ─────→                  ←───── 右侧分组 ─────→
```

##### a. 左侧分组（横向排列，间距 `--space-sm`）

1. **标题**（页面名称）
   - 字号 / 字重：28 / 800（即 `--font-3xl` / `--weight-bold`），颜色 `--color-text-primary`。
   - 文案：与所属一级页面同名 — `今日` / `关心` / `练级` / `收获`。
   - **不随状态变化**：即使切换到历史日（如查看 4 月 19 日数据），标题仍然是模块名"今日"，日期上下文由 § C1 一周日期切换条直接展示，不在顶栏标题里冗余。

2. **戒指连接状态标签**（紧贴标题右侧的 Pill 标签，垂直居中）
   - 圆角 `--radius-pill`，高度 24px，padding `2px 10px`。
   - **无论何种状态，标签整体始终可点击**，统一跳转到《03-个人中心》P6 设备管理页（即"连接戒指页"）。
   - 三种语义状态：

| 状态  | 触发              | 背景                           | 文字颜色                        | 内容                                       | 点击行为                          |
| --- | --------------- | ---------------------------- | --------------------------- | ---------------------------------------- | ----------------------------- |
| 已连接 | 戒指实时在线          | `--color-tag-success-bg`     | `--color-semantic-success`  | `🟢 已连接 · ▭ 85%`（横向电池图标 + 百分比；图标用 CSS 绘制 — 圆角矩形外框 + 右侧正极小凸起 + 内部填充宽度按百分比变化；电量 ≤ 20% 时电池外框、填充、百分比文字一并变 `--color-semantic-warning`） | 跳 P6 进入 P6.1 已连接态（设备详情）       |
| 未连接 | 当前账号未绑定 / 未配对戒指 | `--color-tag-warning-bg`     | `--color-semantic-warning`  | `🟡 未连接`                                  | 跳 P6 进入 P6.2 搜索 / 配对态         |
| 已离线 | 已绑定但失联 ≥ 5 分钟   | `--color-bg-surface-muted`   | `--color-text-tertiary`     | `⚪ 已离线 · N 分钟前`（与 § 9 离线提示条联动）           | 跳 P6 进入 P6.1 设备详情（含重连引导）      |

##### b. 右侧分组（横向排列，间距 `--space-sm`，固定 36×36 圆形按钮）

1. **🌙 主题切换按钮**
   - 浅色模式下显示 🌙；点击 → 切换至深色模式，按钮图标变 ☀️。
   - 深色模式下显示 ☀️；点击 → 切换回浅色模式，按钮图标变 🌙。
   - 状态全局生效（所有一级 / 二级 / 弹层同步）。

2. **👤 个人设置按钮**
   - **未登录**：显示通用 👤 图标（圆形 **白底** `#FFFFFF` + 默认 `--shadow-card` 阴影增强可见性，图标颜色 `--color-text-secondary`）。点击 → 跳《03-个人中心》P1 登录主页。
   - **已登录**：显示用户在个人资料里设置的头像（36px 圆形，无边框）。点击 → 跳《03-个人中心》P5 个人设置页。
   - **头像缺省占位（已登录但用户没传头像）**：用户名首字母 + 品牌渐变底（与 § 7.3 Avatar 默认占位一致）。

##### c. 通用约束

- 顶栏高度 52px（不含状态栏），**背景为白色 70% 透明度**：`rgba(255, 255, 255, 0.7)`，配合 `backdrop-filter: blur(20px) saturate(180%)` 实现 iOS 风格的毛玻璃磨砂效果。深色模式下背景换为 `rgba(20, 22, 25, 0.7)`（即 `rgba` 化的 `--color-bg-overlay-dark`）。
- **iOS 系统状态栏**（顶部 44px，独立组件 `.ios-status-bar`，详见上文）**与一级顶栏共用同一磨砂背景层**——同样的白色 70% 半透明 + 毛玻璃磨砂，视觉上形成连续的"磨砂帽"，从 0 到 96px 一气呵成。**禁止**为状态栏单独使用透明 / 渐变 / 不同颜色。
- 顶栏底部不画分割线，让透明感自然过渡；下方内容滚动时整个"磨砂帽"始终悬浮在最上层。
- 左 / 右两侧分组之间用 `flex: 1` 弹性留白；左侧整体不超过页面宽度的 70%（标题过长截断省略号）。
- **右侧分组永远是、且只是这两个按钮**（🌙 主题 + 👤 个人设置），所有一级页面（今日 / 关心 / 练级 / 收获）保持一致；模块特定的额外动作（关心设置、收获设置等）请下沉到对应页面内部的入口（如设置区段、隐私横幅、列表行），不再放进顶栏。
- Tab Bar 切换不会改变顶栏右侧两个按钮，仅切换左侧标题文案与对应页面背景。

#### 二级页面 `.app-topbar--secondary`

变体：标准型（左返回 + 中标题）、带右侧操作、带副标题。

- 高度：固定 48px，背景白底 `.app-topbar--bar`
- 左：返回箭头 `<`，点击退出当前页（push 转场退栈）
- 中：标题（16 / 600）+ 可选副标题
- 右：可选操作按钮（图标 / 文字）

### 6.2 底部 Tab Bar `.tab-bar`

- 4 个 Tab：今日 ☀️ / 练级 📚 / 关心 ❤️ / 收获 🎁
- 选中态：图标变 fill + 文字主蓝；未选：outline + 文字次灰
- 背景：白底 + `--shadow-tabbar`
- 自动避开 iOS 安全区底部
- Tab Bar 在所有一级页面通用；进入二级页面 / 全屏页面 / 全屏弹层时隐藏

### 6.3 页面容器

- 页面背景：`--color-bg-page` 或 `--bg`（带渐变）
- 内容左右内边距：`--space-lg` (16px)
- 卡片之间纵向间距：`--space-md` (12px)
- 一级页面内容区上下留 `--space-xl` 给状态栏与 Tab Bar

## 8. 弹窗系统

所有弹窗共用 `.modal-overlay`（遮罩层）+ 内容（toast / alert / bottom-sheet / picker-sheet / action-sheet），通过 `.show` 类控制显隐。

### 8.1 Z-index 层级（必须使用 token）

| Token | 数值 | 用途 |
| --- | --- | --- |
| `--z-sheet` | 200 | 普通底部弹层（详情、编辑面板） |
| `--z-picker` | 400 | 滚轮选择器（在 sheet 之上） |
| `--z-alert` | 600 | 确认框 / 轻提示（覆盖滚轮选择器） |
| `--z-critical` | 800 | 关键编辑（个人资料等必须置顶） |

### 8.2 五种弹窗（中英对照）

> 业务文档"形式"列统一用中文术语；本表的英文为代码 / 类名层面的对应关系。

| 中文术语 | 英文（代码层） | 类名 | 用途 |
| --- | --- | --- | --- |
| 轻提示 | Toast | `.modal-overlay.is-toast > .toast` | 操作反馈，1.5s 自动消失，无按钮 |
| 确认框 | Alert | `.modal-overlay > .alert` | iOS 风格确认对话框，宽 270px，必有取消 + 确认 |
| 底部弹层 | Bottom Sheet | `.modal-overlay.is-sheet > .bottom-sheet` | 底部滑入大面板，最高 86vh，可滚动 |
| 滚轮选择器 | Picker | `.modal-overlay > .picker-sheet` | "老虎机"式滚轮选择，行高 36px，scroll snap |
| 操作菜单 | Action Sheet | `.modal-overlay.is-action-sheet > .action-sheet` | iOS 风格底部按钮菜单 |

### 8.3 通用规则

- 底部弹层必须有 `.bottom-sheet-handle` 拖把。
- 点击遮罩关闭轻提示 / 底部弹层（确认框必须按按钮）。
- 同时只显示一个轻提示，间隔 ≥ 500ms。
- 确认框按钮顺序固定：左取消 / 右确认；破坏性右按钮加 `.is-destructive`（红）。
- 滚轮选择器顶部固定：取消（左） / 标题（中） / 完成（右），完成按钮主蓝高亮。

## 9. 全局交互规则

1. **状态切换动画**：进入 / 退出 overlay 200ms ease-out，filter 不打断。
2. **数字增长**：百分比、XP 值入场用 0→目标的 800ms 缓动 + `data-count` 属性驱动。
3. **下拉刷新 / 滑动手势**：仅一级页面顶部允许下拉，二级页面禁用，防误触。
4. **错误反馈**：表单错误用红字 + 抖动 1 次（0.3s），不弹轻提示。
5. **加载态**：按钮内 24px spinner（白），按钮文字保留；列表用骨架屏。
6. **空状态**：emoji 图标 + 标题 + 引导句 + 主 CTA + 次 CTA。

## 10. 隐私 / 数据脱敏规范

| 数据类型 | 默认对外 | 可调 |
| --- | --- | --- |

| 精确位置 | 永不外传 | 仅可显示"约 N 公里" |

## 11. 业务文档写作模板

> 所有模块需求文档（01 / 02 / 03 …）的每个页面 / 弹层 / 子模块必须按以下四段式描述。**不写代码、不写实现细节**，只写"用户看到 / 用户操作什么"，目标是后续可基于本文档直接生成 HTML 原型。

### 11.1 必写四段

#### ① 状态（State）

枚举该模块在不同条件下的形态。如果只有 1 种状态可标注"仅 1 种状态"。

格式：表格三列。

| 状态  | 触发条件 | 关键差异 |
| --- | ---- | ---- |

> 关键差异要写"用户看到的不同"，例如"按钮置灰"、"标题加注日期"、"插入红字提示条"，**禁止**写代码层（如"loading=true"、"添加 .disabled 类"）。

#### ② 元素构成（Element Composition）

按从上到下、从左到右的顺序罗列页面/弹层的视觉元素。每个元素至少标注：位置、内容/数据示例、必要的视觉提示（颜色、字号、对齐）。

复杂页面用编号列表；简单页面用无序列表。子元素用缩进。

允许写：
- 元素类型（按钮 / 输入框 / 列表 / 卡片 / 图标 / 进度条）
- 内容文案 / 数据来源（"动态显示当前用户名" / "示例：8,520 步"）
- 视觉提示（"主蓝高亮" / "渐变蓝紫底" / "24/700 加粗"）
- 与设计 Token 的关联（"使用品牌蓝主色"）
- 复用《00-设计规范》中已有的组件名（按钮 / 卡片 / 底部弹层 / 滚轮选择器…）

不允许写：
- HTML id / CSS class（如 `#xxx`、`.xxx`）
- HTML 属性（`type=tel`、`data-xxx`）
- CSS 变量（`--z-sheet`）
- 框架特定代码

#### ③ 操作逻辑（Interaction Logic）

列出该模块支持的所有用户行为及系统响应。统一格式：

> 用户行为 → 系统响应

至少覆盖：进入 / 退出、点击 / 输入 / 滑动、表单校验、跳转、弹层、轻提示、状态切换。

#### ④ 业务规则（可选）

仅当存在非显而易见的业务约束时才写。比如"60 秒内最多请求 1 次验证码"、"至少保留 1 个挑战可见"、"3 枚戒指上限"。

### 11.2 模板示例

```markdown
## TX. 模块名

一句话用途说明。

### 状态

| 状态  | 触发条件     | 关键差异         |
| --- | -------- | ------------ |
| 默认  | 进入页面     | …            |
| 加载中 | 提交后等待响应  | 主按钮内 spinner |
| 错误  | 服务端返回错误  | 红字 + 抖动 1 次  |

### 元素构成

1. **顶栏**：‹ 返回 + 中标题 + 占位。
2. **主体区**：
   - 卡片 1：…
   - 卡片 2：…
3. **底部按钮**（通栏品牌蓝）："确认"。

### 操作逻辑

- 点击 ‹ 返回 → 回上级页面。
- 点击主体卡片 → 跳 TX.1。
- 点击"确认" → 提交 + 弹轻提示"已保存" + 回首页。

### 业务规则
```

### 11.3 文档结构约定

1. 每个模块文档结构固定为：
   - 文档头（版本、维护人、规范引用、写作模板说明）
