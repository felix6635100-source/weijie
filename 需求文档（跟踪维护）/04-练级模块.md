## 0. 模块概览

「练级」是 App 第二个 Tab（📚），承担"健康挑战游戏化主战场"的核心：让用户在官方 / KOL / 同行者发起的主题挑战中**循序渐进养成长期健康习惯**，并通过等级、徽章、奖品、排行榜、邀请互动构成完整正反馈闭环。

**核心理念**：
> 同样的动作，不同的目的 · 找到你的同行者。

**与今日 / 关心模块的关系**：
- **今日**（每日闭环）：戒指实时数据 + 7 张当日习惯卡。
- **练级**（长期闭环）：用户主动选择的主题挑战、KOL 课程、健康故事，跨周 / 月 / 年。
- **关心**（社交闭环）：好友互动 + 脱敏卡片。

练级与今日的数据是**双向打通**的：用户在练级中加入的挑战，挑战要求的具体行为（步数 / 早睡 / 喝水 / 站立 / 冥想等）会自动对应到今日某张挑战卡的目标计算；今日产生的 XP 累计进练级等级条。

## L1. 练级首页

App 第二个 Tab 的默认页面，承载用户长期健康挑战的浏览、加入与练级进度跟踪。整页**自上而下纵向滚动**，子区段按"先看自己的等级、再看本月推荐、再看自己加入的、最后看更多 / 学习内容"的认知顺序排列。

### 状态

| 状态                | 触发条件                  | 关键差异                                                                                            |
| ----------------- | --------------------- | ----------------------------------------------------------------------------------------------- |
| 已登录·有进行中挑战（默认）    | 已登录 + 已加入 ≥ 1 个挑战     | 完整呈现 8 个子区段；等级条带"连击 N 天"徽章；已加入区段显示进行中挑战列表                                                       |
| 已登录·首次进入（无加入挑战）   | 已登录 + 当前未加入任何挑战       | "已加入挑战"区段隐藏（或替换为提示卡），等级条不显示连击徽章；其他区段正常展示，引导用户先选一个                                              |
| 未登录                | 本地无登录凭据               | 整页内容隐藏，仅显示中心空态（🎮 emoji + 标题"登录后开启你的练级之路" + 一段说明 + 主按钮"立即登录"），点击跳《03-个人中心》P1 登录主页                |

### 元素构成

自上而下：

1. **顶部导航**（一级页面悬浮顶栏，遵循《00-设计规范》§ 6.1 `.app-topbar--primary`）
   - **左侧**：标题"练级"（28/800）+ 戒指连接状态标签（同其它一级页面三态规则）。
   - **右侧**：🔔 消息中心（带未读徽章）+ 👤 个人设置 / 头像（同其它一级页面，v1.2 起；原 🌙 主题切换已下沉至 P5 · 外观）。
   - 顶栏背景：白色 70% 毛玻璃；底部不画分割线。
2. **L1.A 我的练级**（容器，含两个并列入口）：详见 L1.A。
3. **L1.B 本月主推 · 横向轮播**：详见 L1.B。
4. **L1.E 已加入挑战 · 列表**：区段级标题"已加入挑战"放在卡片之外（独立 section header），下方是挑战行列表卡。详见 L1.E。
5. **L1.F 月度挑战 + 目标筛选**：详见 L1.F。
6. **二级入口按钮组**：
   - 通栏卡片"🗂️ 查看全部挑战 · 24 项 ›" → 跳 L3。
   - 通栏粉色卡片"🔗 粘贴朋友的邀请码加入挑战 ›" → 弹 L7。
7. **L1.G 使用入门**：详见 L1.G。区段头与卡片列表左侧内边距统一为 16px。
8. **L1.H 健康故事**：详见 L1.H。区段头与卡片列表左侧内边距统一为 16px。
9. **底部 Tab Bar**（4 Tab，全局通用，"练级"高亮）。

### 操作逻辑

- 顶栏标题"练级" → 不响应（仅展示）。
- 顶栏戒指状态标签 → 跳《03-个人中心》P6 设备管理页（同所有一级页面）。
- 顶栏 🔔 / 👤 → 同所有一级页面（🔔 跳 P7 消息中心；👤 已登录跳 P5 个人设置 / 未登录跳 P1 登录主页）。
- 各区段内点击 → 详见对应子区段。
- 切换底部 Tab → 离开本模块。

### 业务规则

- 整页内容滚动条不显示，但允许触摸 / 滚轮纵向滚动。
- 所有"已加入"标签、连击徽章、已加入挑战的进度数据，与今日模块当日行为实时联动；用户在练级中加入挑战后，今日对应习惯卡会立即出现（若未启用过）或调整目标。
- 隐私：练级页**不展示任何人的具体身体指标数值**；好友 / 排行榜里的"步数"等仅用于挑战完赛排名，受关心圈隐私规则保护。
- L1 首页所有区段（含 L1.G / L1.H 横向滚动卡）左侧起始内边距统一为 **16px**（即 `--space-lg`）；横向卡组超出右屏部分通过滚动浏览，右侧不留对称内边距。

### L1.A 我的练级

L1 首屏第一个内嵌区段，承载"练级进度"与"成就勋章"两个并列入口。区段无统一外框 / 无大标题，由两张并列卡片组成，让用户一眼看到"我现在在哪一级 + 我已经收了多少枚徽章"。

#### 状态

| 状态        | 触发条件               | 关键差异                                                  |
| --------- | ------------------ | ----------------------------------------------------- |
| 默认        | 已登录                | 两个入口正常展示                                              |
| 无连击       | 已登录但断击 ≥ 2 天        | 练级进度入口的"连击 N 天"徽章隐藏                                  |
| 升级中       | 当前 XP 距下一级 ≤ 50 XP | 练级进度入口的进度条末端发光 + 副文"再 N XP 升 LV X+1"                 |
| 无徽章       | 已登录但 0 枚徽章          | 成就勋章入口的副文显示"还没获得勋章 · 完成挑战开始收集"，mini 徽章预览替换为 5 个灰底空圆 |

#### 元素构成

横排 2 区，**左右等宽**（各占可用宽度的 50% - `--space-sm` / 2 间距），高度对齐；也允许根据视觉权重微调比例（如 6:4）但仍保持横排两栏并列。

##### 入口 1 · 练级进度入口（左 / 主入口）

紫粉渐变高亮卡片，承载"我的等级 + 经验进度 + 连击状态"三件信息。

- **左·圆形勋章**：54×54 圆形，半透明白底 + 大 emoji（🏅 默认；后续按用户当前章节切换为 ⭐ / 🏆 / 🦄 等）。
- **右·信息区**（紧凑两栏布局下若空间不足可省略部分元素，优先保留 ① 与 ③）：
  1. 顶行小字（11/800，间距 1px）：`LV 12 · 健康探索者`。
  2. 主标题（14/800）：`{用户名}的练级之路` + 右侧小标签`连击 7 天`（白底 22% 不透明）。
  3. 进度条（高 6px，白底 22%，填充纯白）：当前 LV 经验完成百分比。
  4. 元数据行（11/600）：`累计 1,840 XP · 距 LV 13 还差 660 XP`。

##### 入口 2 · 成就勋章入口（右）

金色淡底卡片，作为徽章墙的入口。

- **左·图标方块**：42×42，圆角 13，金色渐变底 + 🏅 大字。
- **中·主体**：
  - 主文`成就勋章`（14/800）。
  - 副文`已获得 6 枚 · 累计连击 21 天`（11/600，灰）。
  - 横排 5 个 18px 圆形 mini 徽章预览（🔥 / 🌙 / 💧 / 🚶 / 🧘，互相重叠 -6px）。
- **右·箭头 ›**。

#### 操作逻辑

- 点击"练级进度入口"整张卡 → （TBD-1）进入"等级 / XP 详情页"，列出全部解锁等级、进度趋势、徽章。本期暂不实现，点击仅 toast"等级详情建设中"。
- 点击"成就勋章入口"整张卡 → （TBD-3）跳"徽章墙"页面，本期暂不实现，点击 toast"🏅 徽章墙建设中"。
- 练级进度入口的进度条数字增长 0 → 当前值，800ms 缓动入场。

#### 业务规则

- 等级阈值：LV 1 / 5 / 10 / 12 / 15 / 20 / 30 / 50，对应 XP `0 / 500 / 1500 / 2500 / 4000 / 7000 / 12000 / 25000`（TBD-2 待最终敲定）。
- 连击天数从用户首次完成任意挑战日开始连续计算；某天 0 完成则连击清零。
- XP 来源：今日挑战完成 + 关心圈互动 + 加入新挑战首次奖励 + 完赛奖励 + 每日开 App 签到（5 XP，每日上限 1 次）。
- 徽章为系统自动颁发：连续 7 天达标 → 🔥；连续早睡 14 天 → 🌙；累计 100 杯水 → 💧；等等。具体规则后续在《徽章规则.md》中维护（TBD-4）。

### L1.B 本月主推 · 横向轮播

横向滚动卡片轮播，每张卡 86% 屏宽 + scroll-snap，下方有 6 个分页 dot。

#### 状态

| 状态  | 触发条件     | 关键差异                                |
| --- | -------- | ----------------------------------- |
| 默认  | 进入页面     | 默认高亮第 1 张（dot 1 active），其它 5 张右侧排列 |
| 滑动中 | 用户横向滑动   | 当前 snap 到的卡片对应的 dot 高亮              |

> 演示原型固定 6 张主题卡：绵羊咩咩 / 樱花漫步 / 燕成排飞 / 溪边放空 / 一日清泉 / 千江有水。上线后由后端"本月主推池"动态返回 6-8 张。

#### 元素构成

每张卡含：

- **整卡背景**：8 套主题渐变（夜空 / 樱粉 / 林溪 / 跑者落日 / 晨光 / 水蓝 / 呼吸紫 / 站立黄）。
- **左上角日历贴纸**：白底圆角小卡，红色月份缩写`APR`+ 大号日期数字`01`。
- **左中"已加入"小标签**（仅已加入态）：黑色 55% 半透明胶囊 + ✓ 图标 + 文字"已加入"。
- **右上角参与者**：横排 6 个圆形头像（24px，重叠 -6px）+ 下方一行文字`16,176 梦想家` / `5,716 跑者` 等（按挑战类型差异化称谓）。
- **右侧大 emoji 装饰**：透明度 92%，下沉阴影。
- **左下区底部条**：圆形白底大徽章 emoji + 标题（18/800，aspirational 句式，如"绵羊咩咩，数着入眠"）+ 副标题（12/600，时间区间 + 总天数，如"4 月 01 日 - 4 月 30 日 · 30 天"）。

#### 操作逻辑

- 点击整张卡 → 跳 L2 挑战详情。
- 横向滑动 → 切换 snap，dot 联动高亮。

#### 业务规则

- 已加入挑战的卡片左上角强制叠加"已加入"标签，无论用户从哪里加入。
- 排序：① 用户已加入的优先（按加入时间倒序），② 未加入按"距离结束天数升序"。

### L1.E 已加入挑战 · 列表

承载用户当前加入的所有挑战进度概览，作为"快速回到"入口。区段标题"已加入挑战"为**首页级 section header**（与"本月主推"/"月度挑战"等同级），**不放在卡片内部**，与下方的卡片列表分离。

#### 状态

| 状态  | 触发条件      | 关键差异                              |
| --- | --------- | --------------------------------- |
| 默认  | 已加入 ≥ 1 个 | 区段标题 + 卡片列表正常展示，每项含图标 + 标题 + 进度条 + 数值 |
| 无加入 | 已加入 = 0   | **整个 L1.E 区段隐藏**（不显示空态卡，避免冗余）；区段标题与卡片一并隐藏，月度挑战上提 |

#### 元素构成

##### a. 区段头（独立于卡片之外）

- **左**：标题`已加入挑战`（20/800，黑），与"本月主推"/"月度挑战"等同级 section header；左侧内边距 16px，与首页其它区段对齐。可在标题右侧补充小灰字`进行中 · {N} 项`（13/600，灰）作为状态注释。
- **右**：`查看全部 ›` 文字链接（12/700，紫，点击跳 L3）。

##### b. 卡片列表（标题下方独立白底卡）

- **挑战行（每行）**：
  1. **图标方块**：42×42 圆角 13，按挑战主题色渐变 + 大 emoji + 阴影。
  2. **主体**：
     - 第 1 行：挑战标题（14/800，黑）。
     - 第 2 行：`📅 剩余 N 天 · {改善目标简标}`（11/700 紫）。
     - 第 3 行进度条（5px 高 + 主题渐变填充）+ 右侧小字`{当前} / {目标} · {单位}`，达标时附 🏆 emoji。
- 行与行之间**虚线分隔**（border-soft）。

> 演示原型固定 4 项：樱花漫步（92%）/ 绵羊咩咩（76%）/ 一日清泉（60%）/ 2026 心动 200 万步（11%，年度官方挑战）。

#### 操作逻辑

## L6. 邀请好友加入挑战

> **入口**：① L2 已加入态底部"📤 邀请好友 · 已加入" ② L2 未加入态底部"📤 分享给朋友"

承载邀请码生成 + 文案模板 + 复制 + 步骤说明。

### 状态

仅 1 种状态。

### 元素构成

1. **拖把** + 顶部 Header（左 标题`邀请好友加入挑战` + 右 ✕）。
2. **邀请码卡**（粉紫淡渐变底）：
   - 小标签`挑战邀请码`（11/800 玫红）。
   - 大字邀请码`CHL-XXXXXX`（24/800 玫红，字距 3px，等宽字体）。
   - 副文`有效期 24 小时 · 朋友粘贴到 AIRing 即可加入`。
3. **邀请文案预览卡**（白底虚线边）：
   - 完整文案，关键词`邀请码`高亮粉色。
4. **双按钮**：
   - 左白底`复制邀请码`。
   - 右粉紫渐变`📋 复制全文`。
5. **三步骤说明**（白底虚线边）：
   - 1. 点击复制全文 → 黏贴到微信发给朋友。
   - 2. 朋友复制邀请码后打开 AIRing，App 会自动识别剪贴板。
   - 3. 朋友确认 → 加入挑战 → 你们的进度互相可见。

### 操作逻辑

- 点击 ✕ / 遮罩 → 关闭。
- 点击"复制邀请码" → 调系统剪贴板 → toast"📋 邀请码已复制"。
- 点击"📋 复制全文" → 复制完整文案 → toast"📋 邀请文案已复制 · 去微信粘贴"。

### 业务规则

- 邀请码格式：`CHL-XXXXXX`（CHL 前缀 + 6 位大写字母 / 数字，去除易混淆字符 0/O/1/I）。
- 同一用户对同一挑战 24 小时内只生成 1 个邀请码（后端缓存复用）。
- 邀请码全局唯一；过期后自动失效。
- 通过邀请码加入的朋友首次进入挑战时，邀请人 +5 XP（每日上限 50 XP）。

## L7. 粘贴邀请码加入

> **入口**：① L1 月度挑战下方"🔗 粘贴朋友的邀请码加入挑战" ② App 启动时检测剪贴板（与《02-关心模块.md》C3.4 共用同一识别能力）
> **形式**：模态（底部抽屉）
> **关闭去向**：取消 / ✕ / 遮罩 → 关闭

承载手动粘贴邀请码 + 自动识别剪贴板。

### 状态

| 状态     | 触发条件               | 关键差异                                    |
| ------ | ------------------ | --------------------------------------- |
| 默认     | 进入                 | 输入框为空，下方"自动识别"提示                        |
| 已识别    | 用户点击"自动识别"且剪贴板含合法码 | 输入框被自动填入，toast"✅ 已识别邀请码"                |
| 无效输入   | 用户提交但格式不符或码已过期     | 输入框红边 + 抖动 + toast"邀请码无效或已过期"           |
| 加入成功   | 提交合法邀请码            | 关闭模态 + toast"🎉 已加入「{挑战名}」" + 刷新 L1.E   |

### 元素构成

1. **拖把** + 顶部 Header（左 标题`粘贴邀请码` + 右 ✕）。
2. **小提示**：`从朋友消息里复制了 CHL-XXXXXX 之类的码：`
3. **粘贴输入框**（高 60，居中大字 22/800，字距 3px，紫色 focus 边）。
4. **自动识别提示条**（紫色虚线底）：左 📋 + 中`从剪贴板自动识别 / App 启动也会自动识别` + 右`读取 ›`。
5. **双按钮**：
   - 左白底`取消`。
   - 右粉紫渐变`加入挑战`。

### 操作逻辑

- 点击"自动识别 / 读取" → 读剪贴板 → 若匹配 `CHL-[A-Z0-9]{6}` 则填入，toast"✅ 已识别邀请码"；否则 toast"剪贴板里没有有效邀请码"。
- 点击"加入挑战"（输入框非空）→ 校验格式 → 调用加入接口 → 成功则关闭模态 + toast"🎉 已加入「{挑战名}」" + 刷新 L1.E；失败则红边 + 抖动 + 错误 toast。
- 点击"取消" / ✕ / 遮罩 → 关闭。

### 业务规则

- 输入实时大写 + 限制最多 12 字符（`CHL-XXXXXX`）。
- 同一邀请码只能加入 1 次；重复使用提示"已加入该挑战"。
- App 冷启动时若检测到剪贴板含 `CHL-XXXXXX` 模式，弹《02-关心模块》C3.4 同款"自动识别邀请码"浮层（差异：好友码前缀 `AIR-`，挑战码前缀 `CHL-`，互不冲突）。

## 业务规则汇总

### 等级与 XP

- 等级阈值：见 L1.A 业务规则；具体每级 XP 待 TBD-2 敲定。
- XP 来源（综合 4 类）：
  1. **每日行为**：今日挑战完成（每张挑战 +20~+40 XP，详见《01-今日模块》业务规则）。
  2. **挑战互动**：加入新挑战 +10 XP / 邀请朋友加入 +5 XP / 完赛 +50~+200 XP（按挑战级别）。
  3. **关心圈互动**：见《02-关心模块》。
  4. **签到 / 看教程**：每日开 App 首次签到 +5 XP；完整阅读教程 +5 XP / 篇（每天上限 1 篇）。
- 连击规则：连续 N 天有任意挑战完成，算连击 N 天；某天 0 完成则清零。

### 挑战生命周期

- **月度挑战**：每月 1 日 0:00 上架，月末 23:59 结束；下月初按结果颁奖。
- **年度挑战**：1 月 1 日 0:00 上架，12 月 31 日 23:59 结束。
- **本月主推**：编辑团队从月度挑战池中精选 6-8 张推到 L1.B 顶部轮播。
- **退出挑战**：用户随时可退出，已积累的进度数据保留但不再更新。

### 隐私

- 练级**不展示**任何身体指标原始数值（如步数）给其他用户的脱敏卡，仅在**挑战内部排行榜**中按"挑战核心指标"排序展示（仍受用户在关心圈中的"动态显示范围"开关控制）。
- 排行榜可选"全部 / 男生 / 女生"切换；性别字段来自 P5 个人资料，未填则默认归入"全部"，不计入男 / 女分榜。
- 邀请码不暴露用户手机号 / 真实姓名，仅用昵称 + 头像。

### 与今日 / 关心 / 个人中心模块的联动

| 联动场景             | 行为                                                      |
| ---------------- | ------------------------------------------------------- |
| 加入挑战             | 自动启用今日对应挑战卡 + 设置目标（取与原目标的较高值）                            |
| 完成挑战日目标          | 立即在练级 XP 累加；同步刷新 L1.A 等级条 + L1.E 已加入进度                  |
| 退出挑战             | 不删除今日卡的目标，避免"练级退出 → 习惯丢失"                               |
| 关心圈加好友           | 若该好友也加入了同一挑战，挑战详情排行榜中该行追加"好友"角标（TBD-10）                  |
| 个人中心退出登录         | 清空本地练级数据缓存，下次登录从云端拉取                                    |
| 戒指未连接 / 离线       | 练级页可正常浏览所有挑战 / 教程 / 故事；加入挑战不受影响；但**今日数据无法实时计入挑战进度**     |
