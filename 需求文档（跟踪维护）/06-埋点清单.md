# 唯戒 · 一期埋点清单

> 本文档定义一期所有 APP 端需要上报的事件、参数与触发时机，作为前端 / 后端 / 运营三方对齐的统一交付物。
>
> **范围**：仅覆盖运营反馈优化项中需要前端打点的指标（首活动完成率 / 复购率 / 文章阅读率 / 即时反馈 / XP 入口曝光）。其他业务指标（用户注册、设备绑定、签到等）一期沿用已有埋点，不在此重列。
>
> **状态约定**：✅ 已实施 ⏸ 一期不做 / 二期补 🚧 一期需新增

---

## 一、上报规范

### 1.1 通用字段（每个事件都自动带）

| 字段 | 类型 | 来源 | 说明 |
|-----|------|------|------|
| `userId` | string | 登录态 | 未登录用户为 `anonymous` |
| `deviceId` | string | APP | 设备唯一标识 |
| `ts` | int64 | APP | Unix 毫秒时间戳 |
| `appVersion` | string | APP | 客户端版本号 |
| `osPlatform` | string | APP | iOS / Android |
| `osVersion` | string | APP | 系统版本 |
| `sessionId` | string | APP | 单次冷启动会话 ID |
| `source` | string | APP | 触发来源（push / icon / banner / share / deeplink） |

### 1.2 上报通道

- 实时事件 → 走 `/track/event` 单条上报
- 高频事件（≥ 10 次/分）→ 走批量上报队列，每 30 秒或满 50 条触发一次
- 弱网降级 → 落本地队列，下次启动重传，最多保留 7 天

### 1.3 命名规范

- 事件名 `snake_case`，模块前缀 + 动作（如 `activity_join`、`guide_read_finish`）
- 参数名 `camelCase`
- 枚举值用英文小写下划线（如 `difficulty: 'easy' | 'medium' | 'hard'`）

## 二、核心埋点（按运营反馈分组）

### 2.1 活动 · 加入 / 完成（#1 首活动完成率 ≥ 30%）

#### 🚧 `activity_view` · 活动详情曝光

| 时机 | 进入活动详情页（L2）时上报 |
|---|---|
| 用途 | 衡量 Banner / 月度卡的点击曝光转化 |

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `activityId` | int | ✓ | 活动 ID |
| `activityType` | string | ✓ | `banner` / `normal` |
| `difficulty` | string | ✓ | `easy` / `medium` / `hard`（用于分析"难度筛选漏斗"） |
| `entry` | string | ✓ | `banner` / `monthly_card` / `joined_list` / `share` / `push` / `deeplink` |

#### 🚧 `activity_join` · 加入活动

| 时机 | 用户点击「加入挑战」按钮，请求成功后上报 |
|---|---|
| 用途 | 计算"首活动加入"、"复购率（用户加入第二个活动）" |

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `activityId` | int | ✓ | 活动 ID |
| `activityType` | string | ✓ | `banner` / `normal` |
| `difficulty` | string | ✓ | `easy` / `medium` / `hard` |
| `isFirstActivity` | boolean | ✓ | 是否为该用户加入的第 1 个活动（前端通过本地缓存判断；以服务端为准） |
| `joinedActivityCount` | int | ✓ | 加入此活动**前**该用户已参与的活动数 |

**衍生指标**：

- 首活动加入率 = `activity_join (isFirstActivity=true)` 用户数 ÷ 注册用户数
- 复购率 = `activity_join (joinedActivityCount ≥ 1)` 用户数 ÷ 首活动完成用户数

#### 🚧 `activity_progress_update` · 活动进度更新

| 时机 | 用户每次进度变化时上报（注意去重：同一活动同一天最多 1 条） |
|---|---|
| 用途 | 评估活动黏性、绘制"完成分布" |

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `activityId` | int | ✓ | 活动 ID |
| `progress` | int | ✓ | 当前进度百分比（0-100） |
| `progressValue` | int | ✓ | 实际数值（如已达成步数） |
| `progressTarget` | int | ✓ | 目标数值 |
| `daysCompleted` | int | - | 仅 `target_days` 类型有值 |

#### 🚧 `activity_complete` · 完成活动

| 时机 | 服务端判定达成后下发，前端在收到弹窗时上报 |
|---|---|
| 用途 | 计算"首活动完成率"目标（≥ 30%） |

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `activityId` | int | ✓ | 活动 ID |
| `difficulty` | string | ✓ | `easy` / `medium` / `hard` |
| `isFirstActivity` | boolean | ✓ | 是否是该用户完成的第 1 个活动 |
| `durationDays` | int | ✓ | 完成耗时（加入 → 达成的实际天数） |
| `targetDays` | int | - | 仅 `target_days` 类型有值（配置的 minDays） |

**衍生指标**：

- **首活动完成率 = `activity_complete (isFirstActivity=true)` 用户数 ÷ `activity_join (isFirstActivity=true)` 用户数**
- 平均完成耗时 = avg(`durationDays`)，与 `targetDays` 对比

#### 🚧 `activity_quit` · 退出活动

| 时机 | 用户主动点击「退出挑战」时上报 |
|---|---|
| 用途 | 流失分析：哪些活动放弃率最高 |

**参数**：`activityId` / `progressAtQuit`（退出时进度百分比）/ `daysSinceJoin`（加入到退出的天数）

### 2.2 健康指南 · 阅读统计（#4 文章/故事阅读率）

#### 🚧 `guide_view` · 进入文章详情

| 时机 | 文章详情页 onMount 时上报 |
|---|---|
| 用途 | PV 计算（浏览次数）+ UV 去重（阅读人数） |

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `guideId` | int | ✓ | 文章 ID |
| `category` | string | ✓ | 分类（健康科普 / 使用入门 / 健康故事 等） |
| `entry` | string | ✓ | `practice_tab` / `activity_detail` / `share` / `push` |

#### 🚧 `guide_read_finish` · 离开文章详情

| 时机 | 文章详情页 onUnmount 或切到后台时上报 |
|---|---|
| 用途 | 计算"平均停留时长" |

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `guideId` | int | ✓ | 文章 ID |
| `category` | string | ✓ | 同 `guide_view` |
| `duration` | int | ✓ | 本次会话停留秒数 |
| `readPercent` | int | - | 滚动阅读百分比（0-100，可选） |

**计算规则**：

- 平均停留时长 = avg(`duration`) where `duration ≥ 3`
- `duration < 3` 秒视为误点击，**不计入均值**，但仍计入 PV
- 同一用户同一文章多次进入 → PV 累加，UV 仅算 1

#### 🚧 `guide_share` · 文章分享 ⏸（一期不做）

> 一期没有"分享"入口，二期补。

### 2.3 活动即时反馈（#6 进入维度页面动画 + Push）

#### 🚧 `activity_progress_animation_view` · 进度动画曝光

| 时机 | 用户进入今日/挑战详情页，检测到进度增长触发的动画展示后上报 |
|---|---|
| 用途 | 评估"反馈感"是否被用户感知 |

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `activityId` | int | ✓ | 活动 ID |
| `progressDelta` | int | ✓ | 本次进度增长（百分比） |
| `triggerScene` | string | ✓ | `practice_page` / `today_page` / `challenge_detail` |

#### 🚧 `push_notification_received` · Push 触达

| 时机 | 后台 Push 落达时上报（系统级；Android 需通过通知点击补打） |
|---|---|
| 用途 | Push 触达率 + 点击率 |

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `pushId` | string | ✓ | 推送 ID（与推送模板对应） |
| `pushType` | string | ✓ | `activity_goal_reached` / `daily_remind` / `activity_complete` / ... |
| `activityId` | int | - | 仅活动相关 push 有值 |

#### 🚧 `push_notification_click` · Push 点击

| 时机 | 用户点击通知打开 APP 时上报 |
|---|---|
| 用途 | Push 转化漏斗 |

**参数**：与 `push_notification_received` 一致

### 2.4 XP 入口曝光（#7 XP 用途引导）

#### 🚧 `xp_level_page_view` · 我的等级页曝光

| 时机 | 进入"我的等级"页时上报 |
|---|---|
| 用途 | 评估"用户是否知道 XP 能用来做什么" |

**参数**：

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `entry` | string | ✓ | `profile_xp_card` / `xp_toast` / `home_xp_badge` / `deeplink` |
| `currentLevel` | int | ✓ | 当前等级 |
| `totalXp` | int | ✓ | 累计 XP |

#### 🚧 `xp_reward_view` · 等级奖品曝光

| 时机 | 在"我的等级"页滚动到某等级奖品块时上报（曝光去重：同一会话同一奖品仅 1 次） |
|---|---|
| 用途 | 评估"XP 兑换奖品"的诱导效果 |

**参数**：`rewardId` / `levelRequired` / `isUnlocked`

## 三、看板字段对应关系

每个后台字段都标注上报口径，方便后端 ETL 直接对齐：

| 后台字段 | 计算口径 | 依赖事件 |
|---------|---------|---------|
| 活动·参与人数 | distinct `userId` of `activity_join` where `activityId = X` | `activity_join` |
| 活动·完成人数 | distinct `userId` of `activity_complete` where `activityId = X` | `activity_complete` |
| 活动·完成率% | 完成人数 ÷ 参与人数 × 100 | `activity_join` + `activity_complete` |
| 活动·平均耗时（天） | avg `durationDays` of `activity_complete` where `activityId = X` | `activity_complete` |
| 活动·点击次数 | count(`activity_view`) where `activityId = X` | `activity_view` |
| 用户·参与活动列表 | join `activity_join` + `activity_progress_update` + `activity_complete` by `userId` | 三事件 |
| 文章·浏览次数 PV | count(`guide_view`) where `guideId = X` | `guide_view` |
| 文章·阅读人数 UV | distinct `userId` of `guide_view` where `guideId = X` | `guide_view` |
| 文章·平均停留时长 | avg `duration` of `guide_read_finish` where `guideId = X` and `duration ≥ 3` | `guide_read_finish` |
