App 启动后默认的首屏，承载当日健康概览与挑战推进。

| 状态            | 触发条件                  | 关键差异                                                                                                     |
| ------------- | --------------------- | -------------------------------------------------------------------------------------------------------- |

### 元素构成

自上而下 7 个区块：

1. **顶部导航**（一级页面悬浮顶栏，遵循《00-设计规范》§ 6.1 `.app-topbar--primary`）
   - **左侧分组**：
     1. **标题**："今日"（28/800）。**任何状态下都不变**，包括切换到历史日；当前日期上下文由 C1 一周日期切换条直接表达。
     2. **戒指连接状态标签**（紧贴标题右侧 Pill 标签，三态均可点击 → 跳《03-个人中心》P6 设备管理页）：
        - `🟢 已连接 · ▭ 85%`（绿底）— 戒指实时在线 → 跳 P6.1 设备详情。电量图标用 CSS 绘制（圆角矩形 + 右侧正极小凸起 + 按百分比填充内部色块），不用 emoji。
        - `🟡 未连接`（黄底）— 当前账号未绑定戒指 → 跳 P6.2 搜索 / 配对页。
        - `⚪ 已离线 · 7 分钟前`（灰底）— 已绑定但失联 ≥ 5 分钟 → 跳 P6.1 设备详情（含重连引导）。
        - 电量 ≤ 20% 时电池外框、填充、百分比一并变 `--color-semantic-warning`（橙）；标签底色仍保持绿色。
   - **右侧分组**（从左到右两个 36×36 圆形按钮）：
     1. 🔔 **消息中心**（v2.25 起，原 🌙 主题切换已下沉至 P5 · 外观）：常态显示 🔔；右上挂未读徽章（数字 / 纯红点二选一，全 App 一致）。点击 → 跳《03-个人中心》P7 消息中心。
     2. 👤 **个人设置**：
        - 未登录态：显示通用 👤 **白底**图标（`#FFFFFF` + 卡片阴影，与顶栏毛玻璃底形成对比）。点击 → 跳 P1 登录主页。
        - 已登录态：显示用户在个人资料里设置的头像（36px 圆，无边框；用户未传头像则用首字母 + 品牌渐变占位）。点击 → 跳 P5 个人设置。
   - 顶栏背景：白色 70% 透明（`rgba(255, 255, 255, 0.7)`）+ `backdrop-filter: blur(20px) saturate(180%)` 毛玻璃磨砂；高度 52px；悬浮在内容之上，无底部分割线。深色模式下换为深色 70% 透明叠加。
2. **C1 · 一周日期切换**：见 T1.C1。
3. **C2 · 三环活动 / 戒指未连接卡**：见 T1.C2。
4. **C3 · 身体指标 8 卡**：见 T1.C3。
5. **C4 · 今日挑战 7 卡**：见 T1.C4。
6. **C5 · 健康免责声明**：灰色横排小字。
7. **底部 Tab Bar**（4 Tab，全局通用）。

### 操作逻辑

- 顶栏标题"今日" → 不响应（仅展示）。
- 顶栏戒指状态标签 → 跳《03-个人中心》P6 设备详情 / 连接戒指页（按当前状态进入对应子态）。
- 顶栏 🔔 消息中心 → 跳《03-个人中心》P7 消息中心；右上未读徽章在该按钮上同步。原"🌙 / ☀️ 主题切换"已迁移至 P5 · 外观（v2.25）。
- 顶栏 👤 / 头像 → 未登录跳 P1 登录主页；已登录跳 P5 个人设置。
- 一周日期点击 → 详见 T1.C1。
- 三环 / 中心徽章 → 打开 T2。
- 身体指标卡 → 打开 T3；右上"编辑" → 打开 T4。
- 挑战卡 → 打开 T5；右上"编辑" → 打开 T6。
- 喝水 / 跑步 / 吃药卡的 ➕ → 详见 T1.C4。
- 切 Tab → 离开本模块。

### 业务规则

- 戒指实时数据每 60 秒同步一次。
- 历史日模式下"编辑身体指标 / 编辑挑战 / 手动记录"全部禁用，仅查看。
- 顶部页面背景：浅蓝渐变；卡片区背景：白底 + 弱阴影。

### T1.C1. 一周日期切换

横排显示当周一~周日 7 天的"星期标签 + 当日成绩徽章"，**不展示具体日期数字**。视觉焦点集中在两件事：① 过去几天的成绩徽章，② 今天 vs 当前选中日的相对位置。

> 默认演示数据：今天 = **周五**。即过去 4 天（一 / 二 / 三 / 四）显示已获得徽章，今天（五）默认选中，未来 2 天（六 / 日）显示空徽章。

## T7.2. 心肺锻炼实时记录

**底部弹层（半屏，约 70vh）**，浅色底。承载 3 种锻炼类型（跑步 / 骑单车 / 爬楼梯），具体类型由 T6.2 目标设置中"锻炼类型"字段决定，本弹层启动时自动按类型渲染对应字段。

### 状态

| 状态     | 触发条件                       | 关键差异                                            |
| ------ | -------------------------- | ----------------------------------------------- |
| 进行中    | 启动后 / 从暂停态点击"继续"           | 中央暂停按钮                                          |
| 暂停后    | 进行中点击暂停按钮                  | 中央按钮变成两个：左侧"长按结束"红圆环 + 右侧"继续" 绿圆形                |

> 状态独立于锻炼类型；任何类型都有上述 2 态。半屏弹层不再设置"锁屏态"，用户如需锁屏可走系统电源键。

### 元素构成

#### 通用元素（所有锻炼类型共享）

1. **拖把**（顶部圆角小条，提示可下滑收起为浮窗）。
2. **顶部 Header**：状态点（绿色闪烁=进行中 / 黄色=暂停中）+ 状态文字（动态："跑步中" / "骑行中" / "爬楼梯中" / "已暂停"）+ ✕ 收起按钮。
3. **大数行**：当前主要数据（40/800 大字）+ 标签（小字）。**主要数据按类型不同**（详见下表）。
4. **数据网格**：4 个 KPI 单元，**字段按类型不同**（详见下表）。
5. **控制按钮区**（按状态切换）：
   - 进行中：中央暂停按钮（圆形，大）。
   - 暂停后：左 = 长按结束（红圆环 + 提示"长按 1 秒结束"）/ 右 = 继续（绿圆形 ▶）。
6. **轻提示**（顶部，按需）：如"暂停超过 30 分钟将自动结束"。

#### 字段按锻炼类型动态切换

| 锻炼类型  | 大数行                | 数据网格 4 KPI                                 |
| ----- | ------------------ | ----------------------------------------- |
| 跑步    | **总距离 (km)** — 大数  | 总时长 / 实时配速（分'秒"/ km）/ 总消耗（kcal）/ 实时心率（bpm）   |
| 骑单车   | **总距离 (km)** — 大数  | 总时长 / 实时配速（分'秒"/ km）/ 总消耗（kcal）/ 实时心率（bpm）   |
| 爬楼梯   | **总时长 (mm:ss)** — 大数 | 台阶数（级）/ 楼层数（层）/ 总消耗（kcal）/ 实时心率（bpm）       |

> 跑步与骑单车字段相同，仅顶部状态文字、地图轨迹（如有）和"配速"语义略有差异（一个步频、一个踏频）。爬楼梯不再有"距离 / 配速"概念，改为"台阶 / 楼层"。

### 操作逻辑

- 启动时（从 T1 卡 ➕ / T5 hero "▶ 开始记录"）：读取 T6.2 当前"锻炼类型"，按类型渲染对应字段。
- 点击 ✕ 关闭 → **触发 T7.2.1 二次确认弹层**（询问"是否结束本次 [跑步/骑单车/爬楼梯]"），不直接关闭。
- 下滑拖把 → 同上，触发 T7.2.1 二次确认。
- 进行中：点击暂停 → 切换到暂停态。
- 暂停后：点击 ▶ 继续 → 切换回进行中。
- 暂停后：长按红圆环 1 秒 → 也触发 T7.2.1 二次确认（与 ✕ 共用同一确认）。
- T7.2.1 选"取消" → 关闭 T7.2.1，回到 T7.2 当前状态（进行中 / 暂停后均可），**保留计时与已记录数据**。
- T7.2.1 选"结束锻炼" → 持久化本次锻炼数据 → 关闭 T7.2 + T7.2.1 → 弹轻提示"本次 [类型] 已记录"。

### 业务规则

- 后台续记最长支持 N 小时（TBD-3）。
- 锻炼中接电话 / 切到其他 App，计时不中断。
- 用户在锻炼中通过 T6.2 改变"锻炼类型"无效（弹层启动后类型锁定）。
- 关闭意图（✕ / 下滑 / 长按结束）一律走 T7.2.1 确认，避免误触丢失数据。

## 业务规则汇总

### 数据时效

- 戒指实时数据每 60 秒同步一次；离线时显示"X 分钟前"。

- 详情数据**不会**自动同步到关心圈。
- 关心圈展示什么由「个人设置 → 动态发送到关心圈」开关控制（详见《03-个人中心.md》）。
