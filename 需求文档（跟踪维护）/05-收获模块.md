## 0. 模块概览

「收获」是 App 第四个 Tab（🏆），承担"**奖励沉淀展示场**"的核心：让用户每次打开都能完成"我已经收获了这么多"的视觉沉淀感受，形成长期回归动力。

**核心理念**：
> 你走过的每一步、每一夜，都成了你身上的样子。

**与今日 / 练级 / 关心模块的关系**：
- **今日**（每日闭环）：完成行为 → 产生 XP（基础 XP + 等级永久加成 XP）。
- **练级**（长期闭环）：限时活动 / 月度挑战 / 年度挑战完赛 → 产生限定 XP + 限定资产入库。
- **关心**（社交闭环）：用户当前佩戴的称号 / 头像框同步展示在动态卡片上。
- **收获**（成果闭环）：所有奖励资产（等级形象 / IP 皮肤 / 头像框 / 勋章 / 称号 / XP 加成档位）的统一沉淀与佩戴管理场。

**核心闭环路径**：

> 触发层（今日 / 练级产生 XP）→ 沉淀层（XP 累积升级，每 5 级触发进化）→ 表达层（用户在收获 Tab 主动佩戴称号 / 头像框 / 季节皮肤，跨 Tab 同步）。

**与练级模块的边界**：
- 《04-练级模块.md》L1.A 原"我的练级"区段中包含的 **练级进度入口（TBD-1）** 与 **成就勋章入口（TBD-3）** ，本期统一收敛到收获 Tab；练级首页保留入口卡片，但点击行为改为"跳收获 Tab 主页"。详细变更由《04-练级模块.md》后续版本同步（TBD-收获-1）。

## H1. 收获首页

App 第四个 Tab 的默认页面。整页**自上而下纵向滚动**，子区段按"先看自己（IP 形象）→ 再看远方（等级里程碑）→ 再点开看资产柜（我的资产）→ 最后回看昨日辛苦（XP 明细）"的认知顺序排列。

> **入口**：① 底部 Tab 栏 🏆 第四个 Tab ② 升级 / 进化动画结束后跳转 ③ 关心圈动态卡点击佩戴称号区域反向跳入（TBD-收获-2）
> **形式**：一级页面（与今日 / 练级 / 关心同级）
> **关闭去向**：切到其它 Tab

### 状态

| 状态                | 触发条件                  | 关键差异                                                                                     |
| ----------------- | --------------------- | ---------------------------------------------------------------------------------------- |
| 已登录·默认             | 已登录 + 已激活账户            | 完整呈现 4 个子区段（H1.A / H1.B / H1.C / H1.D）；主视觉显示当前等级形象 + 当前激活季节皮肤                            |
| 已登录·新用户首次进入        | 已登录 + 注册 ≤ 24 h        | H1.A 显示雏形阶段形象 + 默认（无季节）皮肤；H1.B 默认定位 L1 节点；H1.C 资产数量显示极小值（称号 1 / 总数、勋章 0 / 总数、头像框 1 / 总数） |
| 已登录·满级（圆满）         | 当前等级 = L25            | H1.A 升级提示文案换为"已达圆满 · 累计 XP {数值}"；进度条满格不流动；H1.B 全部节点已达成                                  |
| 未登录                | 本地无登录凭据               | 整页内容隐藏，进入 H9 未登录引导                                                                       |
| 加载中                | 进入 Tab 后 600ms 内未拿到数据 | 主视觉、里程碑、资产入口、XP 明细统一显示骨架屏                                                                |
| 加载失败               | 接口超时 / 服务异常           | 主视觉显示降级占位（雏形剪影 + "加载失败"），下方区段显示通栏提示"加载失败，请下拉刷新"                                         |

### 元素构成

自上而下：

1. **iOS 系统状态栏**（44px，遵循《00-设计规范》§ 6.1 `.ios-status-bar`）。
2. **顶部导航**（一级页面悬浮顶栏，遵循《00-设计规范》§ 6.1 `.app-topbar--primary`）：
   - **左侧**：标题"收获"（28/800）+ 戒指连接状态标签（同其它一级页面三态规则）。
   - **右侧**：🔔 消息中心（带未读徽章）+ 👤 个人设置 / 头像（v1.9 起；原 🌙 主题切换已下沉至《03-个人中心》P5 · 外观）。
   - 顶栏背景：白色 70% 毛玻璃；底部不画分割线。
3. **H1.A IP 形象主视觉**：详见 H1.A。
4. **H1.B 等级里程碑 · 横滑卡片**：详见 H1.B。
5. **H1.C 我的资产 · 三入口**：详见 H1.C。
6. **H1.D XP 来源记录 · 昨日明细**：详见 H1.D。
7. **底部 Tab Bar**（4 Tab，全局通用，"收获"高亮）。

### 操作逻辑

- 顶栏标题"收获" → 不响应（仅展示）。
- 顶栏戒指状态标签 → 跳《03-个人中心》P6 设备管理页（同所有一级页面）。
- 顶栏 🔔 / 👤 → 同所有一级页面（🔔 跳《03-个人中心》P7 消息中心；👤 已登录跳《03-个人中心》P5 个人设置 / 未登录跳《03-个人中心》P1 登录主页）。
- 各区段内点击 → 详见对应子区段。
- 切换底部 Tab → 离开本模块。
- 下拉到顶 → 触发整页刷新（拉取最新等级 / XP / 资产数量 / 昨日明细）。

### 业务规则

- 整页内容滚动条不显示，但允许触摸 / 滚轮纵向滚动。
- 用户在其它 Tab 完成行为获得 XP 时，下次切换回收获 Tab 自动重新拉取主页数据；本 Tab 不做被动 push。
- 收获 Tab **不承载任何行为入口**（行为入口全部在今日 Tab）；**不承载任何活动入口**（活动入口全部在练级 Tab）。新增模块若涉及"挑战 / 行为 / 活动"，应回看练级 / 今日。
- 隐私：收获 Tab 展示的所有数据均为"用户自身资产"，不涉及好友 / 排行榜数据，与《00-设计规范》§ 10 保持一致。

### H1.A IP 形象主视觉

H1 首屏第一个内嵌区段，承载用户的"身份名片"——当前等级形象 × 当前激活季节皮肤 × 当前佩戴头像框 / 称号。是收获 Tab 视觉沉淀感的核心承载区。

#### 状态

| 状态        | 触发条件                       | 关键差异                                                |
| --------- | -------------------------- | --------------------------------------------------- |
| 默认        | 已登录 + 数据已加载                | 完整呈现等级形象 + 季节皮肤 + 当前佩戴 + 等级数字 + 进度条 + 升级提示文案       |
| 季节皮肤未解锁   | 用户尚未达 L5 / 当前等级未解锁更多季节     | 季节指示器仅亮当前可滑入的季节圆点；尝试滑到锁定季节时显示"L{N} 解锁"占位            |
| 升级临界      | 当前 XP 距下一级 ≤ 50 XP         | 进度条末端发光 + 副文出现"再 N XP 升 LV X+1"                     |
| 圆满（L25）   | 当前等级 = L25                 | 等级形象固定显示"圆满"形态；进度条满格；升级提示文案改为"已达圆满 · 累计 XP {数值}"   |
| 加载中       | 600ms 内未拿到数据               | 形象区显示半透明雏形剪影 + 进度条骨架屏                              |

#### 元素构成

主视觉区域为整宽矩形，高约 380px，纵向自下而上叠加：

##### a. 背景层 · 季节皮肤

- **底层渐变 / 装饰**：根据当前激活季节切换。
  - 默认（无季节，L1-L4）：浅蓝灰渐变，无装饰元素。
  - 春（L5 解锁）：粉色暖渐变 + 樱花飘落动效（飘落速度 ≤ 4 朵/秒）。
  - 夏（L10 解锁）：青绿渐变 + 远处荷塘剪影。
  - 秋（L15 解锁）：橙黄渐变 + 飘落枫叶动效。
  - 冬（L20 解锁）：冷蓝灰渐变 + 雪松剪影 + 雪花动效。
- **季节标签**（左上角）：白底毛玻璃小 chip，11/700，文字"春" / "夏" / "秋" / "冬" / "尚未启程"（无季节态）。

##### b. 主体层 · 中性人物形象

- **居中人物剪影**：高约 220px，PNG / SVG 序列。等级形象由当前等级自动判定，不可主动切换：
  - L1-L4 雏形：抱膝坐姿，轮廓朦胧。
  - L5-L9 觉醒：起身伸展，身边小绿芽。
  - L10-L14 行动：披叶片披风，姿态有力。
  - L15-L19 平衡：花冠加身，盘坐冥想。
  - L20-L24 共生：人物悬浮 + 全身金光 + 背后参天大树虚影。
  - L25 圆满：终极形态（具体视觉由设计另行交付，TBD-收获-3）。
- **人物呼吸动效**：缓慢上下浮动 ±2px，周期 3.6s，永不停止。

##### c. 顶部叠加层 · 当前佩戴

紧贴顶栏下方水平排列的小条：

- **左·用户头像 + 头像框组合**（44×44）：用户头像（圆形）+ 当前佩戴头像框边框包裹；用户未佩戴头像框时仅显示头像。
- **中·用户昵称**（13/700 白色，带阴影）。
- **右·当前佩戴称号**：胶囊小标签（白底 70% 毛玻璃 + 紫字 + 内边距 8px）。

##### d. 底部叠加层 · 等级与进度

紧贴主视觉底部居中：

1. **等级数字**（38/800 渐变文字 `--gradient-xp-gold`）：`LV 12`。
2. **阶段名称**（13/700 灰白）：`行动 · 健康探索者`。
3. **进度条**（高 6px，圆角 999，主题渐变）：当前 LV 经验完成百分比。
4. **进度副文**（11/600 白）：`累计 1,840 XP · 距 LV 13 还差 660 XP`。
5. **升级提示文案**（12/700 白底毛玻璃胶囊）：动态规则见 § 业务规则。

##### e. 季节指示器（底部居中下方）

- 横排 5 个 8px 圆点（默认 / 春 / 夏 / 秋 / 冬）。
- 已解锁圆点：实心彩色（季节色）。
- 未解锁圆点：灰色虚边。
- 当前激活圆点：略大（10px）+ 外发光。

##### f. 右上角 · "档案" 文字链接

- 11/700 白字 + 小 ›，点击跳 H4 等级形象档案。

#### 操作逻辑

- 用户左右滑动主视觉（横向手势，识别阈值 = 屏宽 × 8%）→
  - 滑到已解锁的相邻季节 → 立即切换皮肤（渐变背景 + 装饰动效淡入淡出 200ms），季节指示器同步高亮；切换结果保存为用户当前激活皮肤。
  - 滑到未解锁季节 → 皮肤显示锁定占位（灰底 + 🔒 + "L{N} 解锁"小字），松手回弹到上一可用季节。
- 用户点击"用户头像 + 头像框"组合 → 跳 H3 头像框选择器。
- 用户点击当前佩戴称号胶囊 → 跳 H2 称号选择器。
- 用户点击用户昵称 → 不响应（仅展示）。
- 用户长按主体人物形象（≥ 600ms） → 跳 H4 等级形象档案。
- 用户点击右上角"档案 ›" → 跳 H4 等级形象档案。
- 用户点击进度条 / 副文 → 不响应（仅展示）。
- 等级形象在用户升级到 L5 / L10 / L15 / L20 / L25 时自动切换；切换时机由 H7 进化全屏弹层结束后回到 H1，主视觉淡入新形象。

#### 业务规则

- **等级形象由当前等级自动判定，用户不可主动切换**；切换时机为升级到 5 整数倍的瞬间。
- **季节皮肤为用户主动选择**，系统不会因为升级自动切换皮肤；用户达到 L5 / L10 / L15 / L20 时会通过 H7 进化全屏弹层"提示新季节已解锁"，并提供"立即切换至 {新季节}"按钮供用户选择是否切换。
- **升级提示文案规则**：
  - L1-L4：`再 {距 L5 所需 XP} XP 即可解锁春季皮肤`
  - L5-L9：`再 {距 L10 所需 XP} XP 即可解锁夏季皮肤`
  - L10-L14：`再 {距 L15 所需 XP} XP 即可解锁秋季皮肤`
  - L15-L19：`再 {距 L20 所需 XP} XP 即可解锁冬季皮肤`
  - L20-L24：`再 {距 L25 所需 XP} XP 即可达成圆满`
  - L25：`已达圆满 · 累计 XP {数值}`
- **L20 共生阶段**为质变形态：人物悬浮 + 全身金光 + 背后参天大树虚影；动效不可关闭，作为 25 级旅程的高光时刻。
- **当前佩戴默认值**：用户从未主动佩戴过任何资产时——头像框 = "简约线框"（L1 解锁的基础头像框），称号 = "萌芽者"（L1 解锁的等级称号）。
- **极端情况 · 资产被运营回收**：若用户当前佩戴的资产被运营回收，系统自动切换至上一个佩戴的资产；无该记录时回退到 L1 默认资产。
- **季节指示器圆点数量**：固定 5 个（默认 + 春夏秋冬），即使用户尚未解锁所有季节。

### H1.B 等级里程碑 · 横滑卡片

H1 主视觉下方的等级里程碑滑动卡片区，呈现 25 级全貌。让用户每次进入收获 Tab 都能"远眺一下未来的里程"。

#### 状态

| 状态  | 触发条件         | 关键差异                                            |
| --- | ------------ | ----------------------------------------------- |
| 默认  | 进入 H1        | 默认定位到当前等级节点（屏幕中心 = 当前 LV 卡），左右各显示半张卡作为预告      |
| 滑动中 | 用户横向滑动       | 卡片随手指吸附，吸附阈值 = 卡宽 × 50%；当前吸附的节点对应顶部"等级跳转标识"高亮 |
| 圆满  | 用户已达 L25     | 25 张全部为彩色已达成态；默认定位到 L25                        |

#### 元素构成

##### a. 区段头

- **左**：标题`等级里程碑`（20/800 黑），左侧内边距 16px。
- **右**：跳转链接`查看全部 25 级 ›`（12/700 紫，与右侧 padding 16px 对齐；点击效果同滑到 L1 节点）。

##### b. 横滑卡片列表

- 每张卡 132×168，上下间距 4px，卡片间距 8px，scroll-snap-type: x mandatory。

- **勋章总数与分类**（v1 上线时配置，详细数据见《勋章获取规则.md》）：

- **历史补发**（v1 上线时）：新手引导类 / 身体指标类 / 今日挑战类 / 绝对值里程碑类 / 圆环徽章 支持历史补发；活动徽章默认不做历史补发（运营单独配置）。

- **隐藏勋章**：未解锁前不可访问详情页；解锁后正常展示，包括解锁条件 + 来源说明（系统说明）。
- **跨 Tab 同步**：佩戴生效后同步规则与 H2 一致。

## H6. 普通升级提示

> **形式**：浮层（顶部下滑入场，3.5s 自动消失）
> **关闭去向**：自动消失 / 用户手动点击 ✕ → 回当前 Tab

普通升级（非 5 整数倍等级）使用轻量浮层，避免打断用户当前正在做的事。

### 状态

| 状态  | 触发条件                  | 关键差异                                |
| --- | --------------------- | ----------------------------------- |
| 入场  | 升级事件触发                | 浮层从顶部下滑（200ms ease-out）             |
| 停留  | 入场完成                  | 停留 3.5s                             |
| 退场  | 停留时间到 / 用户点击 ✕        | 浮层向上滑出（200ms ease-out）             |

### 元素构成

横向胶囊浮层（宽 屏宽 - 32px，高 64px，顶部边距 60px，紫粉渐变底，白字）：

- 左·56×56 圆形大数字`LV {N}`（28/800 渐变文字）。
- 中·主行`恭喜升级！`（15/800） + 副行`新等级 · {阶段名}`（11/700）。
- 右·小 ✕（24×24，可点关闭）。

### 操作逻辑

- 升级事件触发 → 浮层从顶部入场。
- 浮层入场后停留 3.5s → 自动退场。
- 用户点击浮层主体（除 ✕ 外） → 跳 H1 收获 Tab + 主视觉跳转。
- 用户点击 ✕ → 立即退场。
- 升级事件连续触发（罕见，跨整数倍）→ 队列展示，间隔 ≥ 500ms。

### 业务规则

- 仅对**非 5 整数倍**等级（L2 / L3 / L4 / L6 / L7 / ... / L24，共 19 级）触发本浮层。
- 5 整数倍升级（L5 / L10 / L15 / L20 / L25）走 H7 进化全屏弹层，不走本浮层。
- 升级后等级数据立即落库，进度条重置为新等级 0%。
- 浮层视觉与《00-设计规范》§ 8 弹窗系统的"轻提示"复用样式 token，非新组件。

## H7. 进化全屏弹层

> **入口**：用户 XP 累积达到 L5 / L10 / L15 / L20 / L25 系统触发
> **形式**：全屏弹层（覆盖当前页，强引导）
> **关闭去向**：用户点击底部主按钮 → 回 H1 收获 Tab + 主视觉切换到新形象

5 整数倍升级是用户 25 级旅程中的"高光时刻"，使用全屏弹层做完整进化动画 + 解锁公告，让用户感受身份跃迁。

### 状态

| 状态           | 触发条件                  | 关键差异                                                |
| ------------ | --------------------- | --------------------------------------------------- |
| 进化中          | 弹层入场后 0-2.5s         | 旧形象淡出 + 新形象淡入 + 阶段名打字机效果 + 背景粒子 / 光效                |
| 公告           | 进化动画完成（≥ 2.5s 后）     | 显示阶段名 + 等级数字 + 解锁奖励列表（IP 皮肤 / 头像框 / 称号 / XP 加成倍率） |
| 圆满（L25）     | 升级到 L25              | 进化动画延长至 4s + 全屏金光 + 文案"你已是身体的同行者"                  |
| 关闭中         | 用户点击主按钮              | 全屏弹层渐隐退出（300ms）                                    |

### 元素构成

全屏深色背景（深紫 → 黑色径向渐变）+ 粒子动效，自上而下：

1. **顶部** 标题`阶段进化`（13/700 白）+ 副文`你的身体在变化`（11/600 灰白）。
2. **中部主视觉区**（高约 60vh）：
   - 旧形象淡出 → 新形象淡入（2.5s 全过程）。
   - 新形象出现后悬浮 + 呼吸动效。
3. **阶段名 + 等级数字**（紧贴主视觉下方）：
   - 阶段名（28/800 渐变文字）：`觉醒` / `行动` / `平衡` / `共生` / `圆满`。
   - 等级数字（13/700 白）：`LV 5` / `LV 10` / `LV 15` / `LV 20` / `LV 25`。
4. **解锁奖励列表**（4 张横排小卡，每张 80×96，间距 8px）：
   - **IP 皮肤卡**：左上 emoji（🌸 春 / 🌿 夏 / 🍂 秋 / ❄️ 冬，L25 无）+ 文字`{季节}皮肤`。
   - **头像框卡**：头像框预览图 + 文字`{头像框名}`。
   - **称号卡**：称号小图标 + 文字`{称号名}`。
   - **XP 加成卡**：⚡ + 文字`XP × {N.NN}`。
5. **L5/10/15/20 专属 · 季节切换提示**（仅这 4 级显示）：
   - 副文`要立即切换到 {新季节} 吗？`（13/700 白）。
   - 单选 chip：`立即切换` / `保持当前`（用户主动选择，默认`立即切换`）。
6. **L25 专属文案**（仅 L25 显示）：
   - 大字（22/800 渐变金）：`你已是身体的同行者`。
   - 副文（13/600 白）：`感谢与唯戒一起走过这 25 级。`
7. **底部主按钮**（通栏紫粉渐变，高 52）：
   - L5 / L10 / L15 / L20：`继续`（17/800 白字）。
   - L25：`走向圆满`（17/800 白字）。

### 操作逻辑

- 升级事件触发 → 全屏弹层入场（500ms 渐入）。
- 进化动画播放（≥ 2.5s，L25 ≥ 4s） → 期间用户点击底部按钮无响应（动画未完成不可跳过）。
- 动画完成 → 底部按钮可点击 + 解锁奖励列表淡入。
- 用户点击`立即切换`/`保持当前`（仅 L5/10/15/20）→ chip 选中态切换；选定值随后跟主按钮一起提交。
- 用户点击底部主按钮 → 弹层退场；H1.A 主视觉切换到新形象 + 用户选择的季节皮肤；解锁的头像框 / 称号入库（不自动佩戴，保留用户手动选择）；XP 加成倍率档位生效，立即作用于后续所有 XP 计算。

### 业务规则

- **触发条件**：用户 XP 累积达到 L5 / L10 / L15 / L20 / L25。每级仅触发 1 次（系统记录已展示标记，避免重复触发）。
- **强引导**：弹层期间禁用 Tab 切换、返回手势；进化动画播放期间不可跳过。
- **季节皮肤切换默认值**：L5/10/15/20 默认`立即切换`；L25 无此选择（无新季节）。
- **奖励入库**：弹层关闭瞬间，4 类奖励均入库到用户账户（IP 皮肤可激活、头像框可佩戴、称号可佩戴、XP 加成倍率立即生效）。
- **断网处理**：弹层期间断网，用户点击主按钮后系统重试 ≤ 3 次，失败时弹轻提示`进化数据保存失败，下次打开 App 自动重试`，弹层关闭。
- **L25 圆满**：用户达到 L25 后，未来再获得 XP 不再升级，但 XP 累计数仍累加（用于 H1.A "累计 XP {数值}" 文案）。

## H8. XP 获取浮层提示

> **入口**：用户在今日 / 练级 / 关心 Tab 完成行为获得 XP 系统触发
> **形式**：浮层（顶部下滑入场，2s 自动消失）
> **关闭去向**：自动消失

每次获得 XP 时显式拆分"基础 XP + 等级加成 XP"，强化用户对等级永久加成的感知。让用户每次获得 XP 都意识到"我升级带来的好处正在生效"。

### 状态

| 状态        | 触发条件               | 关键差异                                  |
| --------- | ------------------ | ------------------------------------- |
| 默认（无加成）   | 用户当前等级 = L1（加成 1.00×）| 仅显示基础 XP，无加成行                       |
| 含加成       | 用户当前等级 ≥ L5         | 显示基础 + 加成两行                          |
| 多事件叠加     | 同一时刻多个行为触发 XP      | 队列展示，间隔 ≥ 500ms                       |

### 元素构成

横向胶囊浮层（宽 屏宽 - 32px，高 56px，顶部边距 60px，金色渐变底，白字）：

- 左·⚡ 大 emoji（28px）。
- 中·主行`+{总计 XP} XP`（17/800 白） + 副行（11/600 白 70%）：
  - 含加成时：`基础 +{基础 XP} · 加成 +{加成 XP}`。
  - 无加成时：`{行为名称}`。
- 右·小`›`（点击跳 H1 主页 H1.D 区段）。

### 操作逻辑

- XP 获取事件触发 → 浮层从顶部入场（200ms）。
- 入场完成 → 停留 2s → 自动退场。
- 用户点击浮层（除 › 外） → 不响应（避免误触跳走）。
- 用户点击 › → 跳 H1 收获 Tab + 自动滚动到 H1.D XP 来源区段。
- 多 XP 事件叠加 → 队列展示，间隔 ≥ 500ms。

### 业务规则

- **触发时机**：行为完成确认（不是开始时）；如挑战完成、活动完赛、互动等。
- **拆分计算公式**：
  - 基础 XP = 行为本身固定 XP（《04-练级模块.md》/《01-今日模块.md》定义）。
  - 加成 XP = 基础 XP × （当前加成倍率 - 1.00），向下取整。
  - 总计 XP = 基础 + 加成。
- **每日上限**：与今日 / 关心 / 练级 模块 XP 上限规则保持一致（如"给好友点赞每日上限 3 次"）。
- **XP 同步落库**：浮层入场前 XP 已落库；浮层仅展示反馈，不阻塞业务。
- **XP 获取浮层不与 H6 / H7 同时显示**：升级事件触发时 H6 / H7 优先级高于 H8，XP 浮层延迟到升级动画结束后展示。
