## 0. 模块概览

「关心」是 App 第三个 Tab（❤️），承担"健康挑战社交化"的核心：让用户与好友、附近同频的人，通过脱敏后的挑战卡片互相看到状态并互动（点赞 / 留言）。关系类型详见 § 业务规则汇总 · 关系类型（v2.5 起统一为陌生人 / 好友 / 特别关心 3 种）。

**核心理念**："你关心的人，和关心你的人。"

## C1. 关心首页

承载脱敏挑战卡片浏览、互动与好友 XP 排行榜入口。

### 状态

| 状态     | 触发条件            | 关键差异                                    |
| ------ | --------------- | --------------------------------------- |
| 未登录    | 本地无登录凭据         | 不显示内容，直接跳转 03-个人中心》P1 登录主页              |
| 已登录无好友 | 已登录但好友数 = 0     | 各子 Tab 内容替换为"还没有好友"空态；排行榜替换为"自己 + 邀请好友" |
| 已登录有好友 | 已登录 + 好友数 ≥ 1   | 默认态：排行榜 + 子 Tab 内容均填充                   |
| 有未读通知  | 好友通知有 ≥ 1 条待我处理 | 我的好友 Tab 标题右侧带红点 + 数字徽章                 |

### 元素构成

自上而下：

1. **顶部导航**（一级页面悬浮顶栏，遵循《00-设计规范》§ 6.1 `.app-topbar--primary`）
2. **TOP 3 排行榜卡片**：详见 C1.1。
3. **3 个子 Tab 切换条**：好友动态 / 发现 / 我的好友，等宽布局，下方滑动指示线。
4. **Tab 内容区**：根据当前选中 Tab 渲染 C1.A / C1.B / C1.C。
5. **底部 Tab Bar**（4 Tab，全局通用）。

### 操作逻辑

- 顶栏标题"关心" → 不响应（仅展示）。
- 顶栏戒指状态标签 → 跳《03-个人中心》P6 设备管理页（按当前状态进入对应子态）。
- 顶栏 🔔 消息中心 → 跳《03-个人中心》P7 消息中心（带未读小红点 / 数字徽章）。原"🌙 / ☀️ 主题切换"已迁移至 P5 · 外观分组（v1.9）。
- 顶栏 👤 / 头像 → 未登录跳 P1 登录主页；已登录跳 P5 个人设置。
- 点击排行榜"查看全部好友排行榜 ›" → 跳 C2。
- 切换子 Tab → 切换内容区。
- 切底部 Tab → 离开本模块。
- 关心设置入口（已下沉到《03-个人中心》P5.care）：① C1.B 隐私横幅的"隐私设置"按钮 → 跳 P5.care · 隐私 / 附近推荐 section；② C1.C 我的好友 Tab 顶部的"💞 关心设置"行（详见 C1.C 元素构成）→ 跳 P5.care。

### C1.1. TOP 3 排行榜卡

横排颁奖台样式，展示前 3 名好友 + 自己的当日 XP。

#### 状态

| 状态     | 触发条件         | 视觉                                |
| ------ | ------------ | --------------------------------- |
| 三人齐    | 好友数 ≥ 3 + 都有 XP | 第 1/2/3 名分别显示对应好友头像 + XP            |
| 缺好友（无好友态） | 好友数 = 0     | 第 1 名 = 自己（头像 + 🏆 + 今日 XP）；第 2/3 名 = "➕ 邀请好友"按钮 |
| 缺好友（仅 1 名好友） | 好友数 = 1   | 缺位的位置变"➕ 邀请好友"按钮                  |

#### 元素构成

1. **顶行**：左侧"今日 TOP 3"小字 + 右侧文字按钮"查看全部好友排行榜 ›"。
2. **三档颁奖台**（横排 3 列，第 1 名居中抬升）：
   - 第 2 名（左）：头像 + 🥈 角标 + 名字 + XP 数值 + "XP"标签。
   - 第 1 名（中，抬升 + 月桂 🌿 装饰）：头像 + 🏆 角标 + 名字 + XP + "冠军 XP"标签。
   - 第 3 名（右）：头像 + 🥉 角标 + 名字 + XP + "XP"标签。
3. **我的位置行**（颁奖台下方独立条）：
   - 左：rank 数字徽章（圆形）。
   - 中：头像 + 名字"你" + 副信息"☀️ 今日已获 +X XP · 点赞 N 次 · 评论 N 次"。
   - 右：总 XP 大数 + "XP"小字。

#### 操作逻辑

- 点击颁奖台头像 → 跳对方个人主页（C5）。
- 点击"➕ 邀请好友"占位 → 跳 C3 添加好友。
- 点击"查看全部好友排行榜 ›" → 跳 C2。
- 点击我的位置行 → 跳 C2（带"我"高亮）。

### C1.A. Tab 1 · 好友动态

#### 状态

| 状态     | 触发条件       | 关键差异                  |
| ------ | ---------- | --------------------- |
| 未登录    | 无登录凭据      | 显示"还没有好友"空态卡 + 添加好友 CTA |
| 已登录无好友 | 已登录但好友数 = 0 | 同上                    |
| 有好友动态  | 好友数 ≥ 1 + 有近期动态 | 时间倒序展示好友动态卡列表          |
| 无近期动态  | 好友数 ≥ 1 但无人发新动态 | 显示空态文案"今天还没有好友动态"      |

#### 元素构成

##### a. 空态卡

- 👋 emoji（大）。
- 标题"还没有好友"（20/700）。
- 副文（两行）：
  - "1. 邀请家人和戒指好友，互相关心、互相点赞，让健康挑战更有动力"
  - "2. 每成功添加一名好友，可获得 **20 XP**"
- 主按钮（通栏品牌蓝）："➕ 添加好友"。

##### b. 好友动态卡（每条）

> v2.4 强一致性要求：每张卡须挂 `data-uid` + `data-feed-type="daily|level"` + `data-trigger="completed|joined|progress-25|progress-50|progress-75|progress-100"` + `data-time="ISO 时间戳"`，并按 `data-time` 倒序渲染。详见 § 业务规则 · 动态发送机制。

1. **卡上方提示行** `dyn-time-hint`：灰字小条（如"今早 06:30 · 今日挑战 · 完成" / "3 天前 · 练级挑战 · 加入"），独立于卡片之外，提示该动态的发送时机与类型。
2. **头部**（横排）：
   - 左：头像（带 R 戒指角标）。
   - 中：名字（16/700）+ （若是特别关心）`🔔 特别关心` 标签 + 副信息（同城 / 异地 / 连续 N 天 / "今日第 N 项"）。新好友前 7 天内额外显示"新加入"灰底小标签。（v2.5：原"亲友 / 关注的好友"关系标签已取消，因为出现在 C1.A 流里的都是好友）。
   - 右：时间戳（"今日" / "昨夜" / "刚刚" / 具体时间）。
3. **挑战块**（带渐变底，按挑战类型不同色）：
   - 顶行：大图标（圆形）+ 挑战名 + 状态徽章（已完成 / 进行中）。
   - 中行：进度条 + 百分比（彩色加粗）。
   - 底行：一句话脱敏文案（如"今天坚持走完 8,000 步目标 · 比昨天多坚持了 15 分钟"）。
4. **互动行**（v2.4 校正：朋友圈式合并行，替代原 v2.3 "操作栏 + 评论区" 两层结构）：
   - **第 1 行 · 点赞行**：❤️ 点赞按钮 + 点赞名单（"林雨晴、陈乐 等 8 个赞"，名字可点击跳 C5）+ 右侧"💬 评论"按钮。**已点赞态**：心形填红 + 名单首位插入"我"；**未点赞态**：心形描边灰；XP 提示由 toast"+3 XP"承担，不再挂在按钮旁。
   - **第 2 行起 · 评论列表**：每条 = 昵称（"我"加粗 / 别名带 `回复 XX` 二级缩进）+ "："+ 内容；点击单条评论 = 引用回复（光标进入内联输入框，前缀"@对方名字 "）。
   - **第 N 行 · 内联输入框**（点击"💬 评论"展开）：单行文本框（占位"仅对方可见 · 写一句关心的话"，最多 80 字）+ 右侧"发送"按钮。发送后追加到列表底部。
   - 整行点赞 / 评论数 ≥ 0 都按此结构展示；0 互动时"点赞名单"位文案改为引导文（如"还没有人点赞，去给 TA 一个鼓励吧 👏"）。

#### 操作逻辑

- 点击头像 → 跳对方个人主页（C5）。

### C3.4. 自动识别邀请码（v2.4 重写 · 拆为两步流程）

> **v2.4 变更（与 HTML 原型对齐）**：原 v2.3 把 C3.4 定义为单一弹窗（含完整邀请码 + 取消 / 确认按钮），HTML 实际拆成了 **"过渡识别页 + 复用 C3.2.a 二次确认弹窗"** 两步流程，体验更平滑。本节同步拆分。

#### 触发场景

用户在外部（如微信聊天 / 朋友圈 / 短信）复制了 AIRing 邀请码，再次打开 App 时，前台自动检测剪贴板，依次走 C3.4 → C3.2.a。

#### 步骤 1 · 过渡识别页（C3.4 主体）

**形式**：全屏过渡页 🟦（覆盖 App 主内容，无 Tab Bar）；时长固定 **500ms**，超时自动跳步骤 2 或失败兜底。

##### 状态

| 状态  | 触发条件              | 关键差异              |
| --- | ----------------- | ----------------- |
| 识别中 | App 启动 / 切回前台时检测到剪贴板有合法格式邀请码（`AIR-XXXXXX`）| 显示 💍 大图标 + "正在识别邀请码" 标题 + "剪贴板检测到 AIR-XXXXXX" 副文 + 3 点 loading 动画 |
| 失败兜底 | 500ms 内查不到对应用户 / 邀请码已过期 | 该过渡页 toast"邀请码无效或已过期"，不进入步骤 2 |

##### 元素构成

1. **💍 大图标**（居中）：48-64px，淡蓝光晕背景。
2. **标题**："正在识别邀请码"（17/700，居中）。
3. **副文**："剪贴板检测到 **AIR-XXXXXX**"（13/500，灰字，居中；邀请码加粗显示）。
4. **加载动画**：3 个圆点的依次淡入淡出。

##### 操作逻辑

- 进入页面即开始解析；**不接受任何用户操作**（用户不可点 / 滑），仅作过渡视觉。
- 500ms 后自动转步骤 2 弹起 C3.2.a。
- 失败时显示 toast 后回到关心 Tab，**不打扰当前任务**。

#### 步骤 2 · 添加好友二次确认（复用 C3.2.a）

**形式**：modal 居中弹窗（与"输入邀请码后点'添加'"走同一个 C3.2.a），叠加在关心 Tab 之上。

##### 元素构成

详见 [C3.2.a 添加好友二次确认](#c32a-添加好友二次确认)。从 C3.4 进入时：
- 头像 = 该邀请码对应用户的真实头像（带 R 角标）。
- 主文 "是否添加 **{昵称}** 为好友？"
- **来源说明（`src` 字段）= "通过剪贴板邀请码 AIR-XXXXXX"**（与"通过昵称搜索 / 通过邀请码"互斥）。

##### 操作逻辑

- 点击"取消" → 关闭弹窗，回到关心 Tab；剪贴板不清空（用户可能要复制别的内容）。
- 点击"确认添加" → 弹轻提示"添加好友成功，可以互相查看动态了"；C1.C 好友列表实时增加该好友；剪贴板内的邀请码标记为已处理（同次 session 内不再重复弹）。

#### 业务规则

- 邀请码格式校验：`^AIR-[A-Z0-9]{6}$`，不符合的剪贴板内容**不触发** C3.4。
- 同一邀请码本次 App session 内**仅识别 1 次**（避免反复弹弹窗）；下次冷启动后再次复制同一码会重新触发。
- C3.4 与 P1 登录主页冲突时（用户未登录）：先走登录流程，登录成功后再触发 C3.4。
- 失败兜底场景（详见 § 步骤 1 状态表）的 toast 文案待与算法侧对齐：邀请码无效 / 邀请码已过期 / 已是好友 / 无法添加自己 — 见 C-TBD-2。

| **特别关心** | 在"好友"基础上，用户在 **C5.1 主页设置** 中给 TA 打"特别关心"标记                  | 与好友一致 + ① 三处显示 `🔔 特别关心` 标签（C1.A 昵称旁 / C1.C 顶部独立分组 / C5 名牌旁）；② C1.C 列表内原行隐藏（仅在"特别关心"分组里出现一次） | TA **发新动态时**触发系统通知栏推送（App 未在前台也能收到），与普通点赞 / 评论通知独立 |

> **核心理念**：好友是**互相对等**的关系（你看我 = 我看你）；特别关心是**单向标记**（我把 TA 设为特别关心 ≠ TA 也把我设为特别关心），只影响推送强度。
>
> **"加为好友"是全局唯一术语**（v2.5 起）：所有原"申请关注 / 关注的好友 / 亲友"措辞统一改为 **"加为好友 / 发送好友请求 / 好友"**，避免双套文案造成认知负担。

### XP 经济

| 给他人留言    | +5  | 每日 30   |

### 隐私铁律

1. 任何人都看不到原始数值（步数、心率、深睡时长等具体数）。
2. 关心圈仅展示「挑战完成 / 进行中」状态 + 脱敏描述。
3. 用户可在 P5.care 按挑战项控制"好友 / 陌生人"两档可见性（陌生人态默认全不可见，好友态可逐项开关）。
4. 药品 / 体重 / 健康状况强私密，默认关闭。
5. 位置信息精度最小 1 km，永不暴露精确位置。
6. 排行榜每周一凌晨重置今日 / 本周 XP，但累计 XP 永不重置。

### 通知规则

- 收到点赞 / 评论 / 好友请求 → 推送到锁屏（在系统通知中开启）。
- 通知频率：合并 5 分钟内的同人多次互动为 1 条。
- 用户可在《03-个人中心》P5.care · 关心通知 关闭"接收好友点赞通知"（v2.4 起，原"C6 组 5"已闭环到 P5.care；P5.care 当前 5 个 section 已覆盖；若"接收点赞通知"开关尚未在 P5.care 落位，见 C-TBD-8）。

### 特别关心机制（v2.4 新增 · v2.5 校正）

> 在 3 种关系类型（陌生人 / 好友 / 特别关心）中，"特别关心"是叠加在"好友"之上的**单向标记**。仅影响"通知强度 + 信息呈现优先级"，不改变可见内容档位（陌生人对应有限可见 / 好友对应完全可见，由 P5.care 控制）。

#### 触发与持久化

- 唯一开关入口：**C5.1 主页设置（特别关心）**（从 C5 顶栏 ⚙ 进入）。
- 开关切换实时持久化，无需"保存"。

#### 标记的呈现

被设为"特别关心"的好友，在以下 3 处一致出现 `🔔 特别关心` 小标签：

1. **C1.A 好友动态卡**：昵称右侧标签。
2. **C5 个人主页 Profile Hero**：名字右侧标签。
3. **C1.C 我的好友 Tab**：进入"特别关心"独立分组（标签隐式表达，分组本身即为标记）。

#### 我的好友列表的"分组 + 去重"规则

- C1.C 顶部"特别关心"分组**仅当组内至少有 1 位好友时显示**。
- 同一好友**不在两处重复出现**：被设为"特别关心"的好友，会从下方"我的好友"列表中隐藏原行，仅在"特别关心"分组中呈现一次。
- 取消"特别关心"后，原行立即回到"我的好友"列表，分组若为空则整段隐藏。

#### 系统通知

- 开启后，TA **发布新动态时**触发 App 系统通知（标题"🔔 [TA 名] 刚发布了新动态"+ 副文挑战名 + 完成 / 加入 / 里程碑标签）。
- 系统通知不依赖锁屏开关；但用户可在 iOS 系统设置里关掉 App 通知（与点赞 / 评论通知共用同一通道，下行不区分级别）。
- 同一好友 5 分钟内多条动态合并为 1 条通知（与 § 通知规则 一致）。
- 关掉特别关心后立即停止推送，已下发的通知不撤回。

#### 上限

- 单用户最多可设 **20 位**特别关心好友（避免噪声）；达上限时再切开关，弹轻提示"最多 20 位 · 请先取消其他特别关心"并阻止操作。

#### 与 P5.care 的关系

- "特别关心"是**针对单个好友**的设置，归 C5.1；P5.care 是**全局开关**（动态发送 / 谁可以看 / 通知偏好），两者解耦。
- 若用户在 P5.care 关闭"接收好友点赞通知"，**不影响**特别关心的"发布动态推送"（两个通道独立）。

### 动态发送机制（v2.3 新增）

挑战分两类，发动态的时机不同：

#### 今日挑战（每日重复任务）

- **触发**：当日挑战**完成时**发送 1 条动态。
- **典型项**：走路 / 睡眠 / 喝水 / 跑步 / 站立 / 吃药 / 消耗。
- **理由**：今日挑战每天循环、无"开始时间"，且部分由戒指被动记录（如步数 / 心率），因此"完成时炫耀达成"是最自然的发送时机。未完成不发动态，避免噪声。

#### 练级挑战（长期主动加入）

- **触发**：分 5 个里程碑发送动态。

  | 触发时机 | `data-trigger` 值 | 文案语义 |
  | --- | --- | --- |
  | 加入挑战 | `joined` | "我加入了 X，期待 N 天后" |
  | 进度 25% | `progress-25` | "X 已完成 25% 里程碑" |
  | 进度 50% | `progress-50` | "X 已完成一半，节奏稳定" |
  | 进度 75% | `progress-75` | "X 已完成 75%，再坚持冲刺" |
  | 进度 100%（完赛） | `progress-100` | "X 完赛！等待抽奖" |

- **典型项**：30 天早睡早起 / 樱花漫步 30 天 50 公里 / 年度大挑战 等。
- **理由**：练级挑战是长周期 + 主动报名，需要在关键里程碑给出持续的社交反馈，维持热度并邀请好友互动。
- **去重**：同一挑战的同一里程碑只发 1 次（即使中途回退后再到达，也不重复发）。

#### 字段约定（C-Me 我的动态 / C5 好友主页 / C1.A 好友动态 共用）

每张动态卡 `.dyn-card`（或 `.my-dyn-card`）必须带：

- `data-feed-type="daily|level"`：动态类型。
- `data-trigger="completed|joined|progress-25|progress-50|progress-75|progress-100"`：触发原因。
- `data-time="ISO 时间戳"`：动态发送时间，列表按其倒序排（最新在最上）。
- 卡内副文行（`.dyn-time`）展示语义化时间 + 类型 + 触发标签，如：「今天 12 分钟前 · 今日挑战 · 完成」/「3 天前 · 练级挑战 · 加入」/「今天 09:42 · 练级挑战 · 50% 进度」。

#### 列表排序

C-Me 我的动态、C5 好友主页 TA 的动态、C1.A 好友动态（首页流）三处列表均按 `data-time` 倒序展示，无分页时一次性渲染。
