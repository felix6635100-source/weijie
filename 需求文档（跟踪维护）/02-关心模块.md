## 0. 模块概览

「关心」是 App 第三个 Tab（❤️），承担"健康挑战社交化"的核心：让用户与亲友、关注的好友、附近同频的人，通过脱敏后的挑战卡片互相看到状态并互动（点赞 / 留言）。

**核心理念**："你关心的人，和关心你的人。"

## C1. 关心首页

承载脱敏挑战卡片浏览、互动与好友 XP 排行榜入口。

### 状态

| 状态     | 触发条件            | 关键差异                                    |
| ------ | --------------- | --------------------------------------- |
| 未登录    | 本地无登录凭据         | 不显示内容，直接跳转 03-个人中心》P1 登录主页              |
| 已登录无好友 | 已登录但好友数 = 0     | 各子 Tab 内容替换为"还没有好友"空态；排行榜替换为"自己 + 邀请好友" |
| 已登录有好友 | 已登录 + 好友数 ≥ 1   | 默认态：排行榜 + 子 Tab 内容均填充                   |
| 有未读通知  | 好友通知有 ≥ 1 条待我处理 | 我的好友 Tab 标题右侧带红点 + 数字徽章                 |

### 元素构成

自上而下：

1. **顶部导航**（一级页面悬浮顶栏，遵循《00-设计规范》§ 6.1 `.app-topbar--primary`）
2. **TOP 3 排行榜卡片**：详见 C1.1。
3. **3 个子 Tab 切换条**：好友动态 / 发现 / 我的好友，等宽布局，下方滑动指示线。
4. **Tab 内容区**：根据当前选中 Tab 渲染 C1.A / C1.B / C1.C。
5. **底部 Tab Bar**（4 Tab，全局通用）。

### 操作逻辑

- 顶栏标题"关心" → 不响应（仅展示）。
- 顶栏戒指状态标签 → 跳《03-个人中心》P6 设备管理页（按当前状态进入对应子态）。
- 顶栏 🔔 消息中心 → 跳《03-个人中心》P7 消息中心（带未读小红点 / 数字徽章）。原"🌙 / ☀️ 主题切换"已迁移至 P5 · 外观分组（v1.9）。
- 顶栏 👤 / 头像 → 未登录跳 P1 登录主页；已登录跳 P5 个人设置。
- 点击排行榜"查看全部好友排行榜 ›" → 跳 C2。
- 切换子 Tab → 切换内容区。
- 切底部 Tab → 离开本模块。
- C6 关心设置入口：① C1.B 隐私横幅的"隐私"链接；② C1.C 我的好友 Tab 顶部的"关心设置 ⚙"行（详见 C1.C 元素构成）。

### C1.1. TOP 3 排行榜卡

横排颁奖台样式，展示前 3 名好友 + 自己的当日 XP。

#### 状态

| 状态     | 触发条件         | 视觉                                |
| ------ | ------------ | --------------------------------- |
| 三人齐    | 好友数 ≥ 3 + 都有 XP | 第 1/2/3 名分别显示对应好友头像 + XP            |
| 缺好友（无好友态） | 好友数 = 0     | 第 1 名 = 自己（头像 + 🏆 + 今日 XP）；第 2/3 名 = "➕ 邀请好友"按钮 |
| 缺好友（仅 1 名好友） | 好友数 = 1   | 缺位的位置变"➕ 邀请好友"按钮                  |

#### 元素构成

1. **顶行**：左侧"今日 TOP 3"小字 + 右侧文字按钮"查看全部好友排行榜 ›"。
2. **三档颁奖台**（横排 3 列，第 1 名居中抬升）：
   - 第 2 名（左）：头像 + 🥈 角标 + 名字 + XP 数值 + "XP"标签。
   - 第 1 名（中，抬升 + 月桂 🌿 装饰）：头像 + 🏆 角标 + 名字 + XP + "冠军 XP"标签。
   - 第 3 名（右）：头像 + 🥉 角标 + 名字 + XP + "XP"标签。
3. **我的位置行**（颁奖台下方独立条）：
   - 左：rank 数字徽章（圆形）。
   - 中：头像 + 名字"你" + 副信息"☀️ 今日已获 +X XP · 点赞 N 次 · 评论 N 次"。
   - 右：总 XP 大数 + "XP"小字。

#### 操作逻辑

- 点击颁奖台头像 → 跳对方个人主页（C5）。
- 点击"➕ 邀请好友"占位 → 跳 C3 添加好友。
- 点击"查看全部好友排行榜 ›" → 跳 C2。
- 点击我的位置行 → 跳 C2（带"我"高亮）。

### C1.A. Tab 1 · 好友动态

#### 状态

| 状态     | 触发条件       | 关键差异                  |
| ------ | ---------- | --------------------- |
| 未登录    | 无登录凭据      | 显示"还没有好友"空态卡 + 添加好友 CTA |
| 已登录无好友 | 已登录但好友数 = 0 | 同上                    |
| 有好友动态  | 好友数 ≥ 1 + 有近期动态 | 时间倒序展示好友动态卡列表          |
| 无近期动态  | 好友数 ≥ 1 但无人发新动态 | 显示空态文案"今天还没有好友动态"      |

#### 元素构成

##### a. 空态卡

- 👋 emoji（大）。
- 标题"还没有好友"（20/700）。
- 副文（两行）：
  - "1. 邀请家人和戒指好友，互相关心、互相点赞，让健康挑战更有动力"
  - "2. 每成功添加一名好友，可获得 **20 XP**"
- 主按钮（通栏品牌蓝）："➕ 添加好友"。

##### b. 好友动态卡（每条）

1. **头部**（横排）：
   - 左：头像（带 R 戒指角标）。
   - 中：名字（16/700）+ 关系标签（亲友 / 关注的好友）+ 副信息（同城 / 异地 / 连续 N 天 / "今日第 N 项"）。
   - 右：时间戳（"今日" / "昨夜" / "刚刚" / 具体时间）。
2. **挑战块**（带渐变底，按挑战类型不同色）：
   - 顶行：大图标（圆形）+ 挑战名 + 状态徽章（已完成 / 进行中）。
   - 中行：进度条 + 百分比（彩色加粗）。
   - 底行：一句话脱敏文案（如"今天坚持走完 8,000 步目标 · 比昨天多坚持了 15 分钟"）。
3. **操作栏**（横排）：
   - ❤️ 点赞按钮（含 +3 XP 标签 / 已点赞态变红色 + "已点赞"）。
   - 💬 点评论按钮。
   - 灰字"· N 人点赞"。
4. **评论区**（点击评论展开）：
   - 已有评论列表：每条 = 昵称（"我"加粗）+ 内容 + 时间戳 + 送达状态。
   - 输入区：单行文本框（占位"仅对方可见 · 写一句关心的话"）+ 右侧"发送"按钮。

#### 操作逻辑

- 点击头像 → 跳对方个人主页（C5）。
- 点击 ❤️ → 切换为已点赞态（红心 + "已点赞" + +3 XP）；总点赞数 +1；XP 计入今日中环。
- 点击 💬 → 展开 / 收起评论区。
- 点击评论条 → 引用回复（光标进入输入框，前缀"@对方名字 "）。
- 点击"发送" → 追加留言到列表底部，输入框清空。
- 点击 C1.A 内的 "➕ 添加好友"按钮 → 跳 C3。

#### 业务规则

- 同一用户对同一挑战仅能点赞 1 次；再次点击为取消，XP 同步扣除。
- 点赞每日上限 60 XP（在 C6 组 5 调整）。
- 评论每日上限 30 XP，每条 +5 XP。
- 同一好友多个挑战会展示成多张独立卡，按时间倒序。

### C1.B. Tab 2 · 发现

基于共同挑战与健康节奏推荐附近 / 同频用户。

#### 状态
只有一种状态，登录后，系统默认会给用户推荐附近的好友
#### 元素构成
1. **隐私横幅**：使用 蓝色 7.9 通告栏 ，文案例“可能跟你在同一个城市 · 互不精确定位”，右侧有按钮“隐私设置”，点击前往 个人设置页
2. **页面内标题**：身边同频的人" + 右侧紫底徽章"今日推荐 3 人"
3. **推荐卡（每条）**：
	1. **头部**（横排）：
		1. 左：头像（带 R 角标）。
		2. 中：名字 + 性别 + 副信息（年龄 + 可能同城 / 异地 + 一句标签）。
		3. 右：距离（如"约 10 公里"）。
	2. **标签行**（横向滚动）：✨ 共同挑战数 + 标签（晨跑党 / 早睡 22:30 / VO2 Max 48 等）。
	3. **当前活动卡**：彩色图标 + 一句话（如"正在完成「激活心肺循环」 · 已完成 72% · 此刻就在你附近"）。
	4. **操作栏**（双按钮）：
		1. 申请关注 +10 XP（主按钮）。
		2. 忽略（次按钮）。
4. **查看更多**：居中圆角按钮"↓ 查看更多"，点击再加载 3 个卡片

#### 操作逻辑

- 点击空态卡"登录账号" → 跳 P1 登录。
- 点击"隐私"链接 → 跳 C6。
- 点击推荐卡头像 → 跳对方个人主页（脱敏视图，C5）。
- 点击"申请关注 +10 XP" → 卡片飞出动画 + 发送好友请求（对方在 C4 待我同意中可见）+ 轻提示"已申请关注，等待对方同意"。
- 点击"忽略" → 卡片淡出消失，本周不再推荐该人。
- 点击"↓ 查看更多" → 加载下一组 3 人。

#### 业务规则

- 申请关注每日上限 50 XP（5 次）。
- 推荐池排除"已是好友 / 已申请待对方同意 / 已被对方忽略"的用户。
- 距离精度最小 1 km，永不暴露精确位置。

### C1.C. Tab 3 · 我的好友

好友总览与添加好友 / 好友通知二级入口。

#### 状态

| 状态     | 触发条件     | 关键差异                |
| ------ | -------- | ------------------- |
| 无好友    | 好友数 = 0  | 列表区域显示"还没有好友"空态卡    |
| 有好友    | 好友数 ≥ 1  | 显示好友列表 + 邀请激励空态     |
| 有未读通知  | 待我同意 ≥ 1 条 | "好友通知"入口右上角红点 + 数字 |

#### 元素构成

1.  **二级入口区**（横排两个卡）：
   - 添加好友卡：紫色图标 👥 + 主文"添加好友" + 副文"分享链接 · 搜索昵称" + ›。
   - 好友通知卡：红色图标 🔔（带未读数小红圈）+ 主文"好友通知" + 副文（动态："2 条待处理"）+ ›。
2. **我的好友列表**：
   - 区段头：左侧标题"我的好友" + 右侧紫底徽章"共 N 位"。
   - 副文"点击头像查看 TA 的主页与历史动态"。
   - 列表（每行）：头像 + 名字 + 关系标签 + 副信息（同城 / 异地 / 连续 N 天达标 / 未佩戴 Ring）+ ›。
3. **邀请激励空态**（列表底部）：
   - 💌 emoji。
   - 标题"每成功添加一位亲友 +20 XP"。
   - 副文"解锁「关心使者」徽章 · 还差 N 位"。

#### 操作逻辑

- 点击"关心设置"行 → 跳 C6 关心设置。
- 点击"添加好友"卡 → 跳 C3。
- 点击"好友通知"卡 → 跳 C4；进入后清空未读红点。
- 点击好友行（头像或任意位置） → 跳 C5 个人主页。

## C2. 好友 XP 排行榜

二级页面，列出所有好友按 XP 排序，含自己。

### 状态

| 状态  | 触发条件      | 关键差异         |
| --- | --------- | ------------ |
| 无好友 | 好友数 = 0   | 仅显示自己一行 + 引导添加 |
| 有好友 | 好友数 ≥ 1   | 自己置顶 + 完整列表  |

### 元素构成

1. **顶栏**（二级页面顶栏）：‹ 返回 + 中标题"好友排行榜" + 占位。
2. **我的排名置顶汇总**（高亮卡）：
   - 左：rank 圆形大徽章。
   - 中：头像 + 名字"你"（带"我"小标签）+ 副信息"☀️ 今日已获 +X XP · 点赞 N 次 · 评论 N 次"。
   - 右：总 XP 大数 + "XP"小字。
3. **完整列表**（按 XP 降序）：
   - 每行：rank 数字 + 头像 + 名字 + 今日 +X XP（彩色）+ 总 XP（右侧）。
4. **底部说明**："关心积分由点赞、评论、关注、邀请等互动累计 · 每周一凌晨重置"。

### 操作逻辑

- 点击 ‹ 返回 → 回 C1。
- 点击列表中的好友行 → 跳 C5 对方个人主页。
- 滚动到底部 → 不分页（一次性加载，因好友数有限）。

### 业务规则

- 排行榜每周一凌晨重置今日 / 本周 XP，但累计 XP 永不重置。
- 仅展示已互相同意的好友。

## C3. 添加好友

二级页面，承载 3 种添加方式入口。

### 状态

仅 1 种状态。

### 元素构成

1. **顶栏**：‹ 返回 + 中标题"邀请好友"。
2. **预览卡**（白底高亮卡）：
	1. 头像 + 名字 + AIRING 徽章。
	2. 邀请文案预览（示例）："💞 复制我的邀请码 AIR-XXXXXX，打开AIRing APP 成为我的 AIRING 好友，一起养成健康习惯！"
3. **渠道选择标题**："选择邀请渠道"。
4. **3 个渠道入口**（横排 3 列,都是白色卡片）： 
	1. 🔗 复制链接

## 业务规则汇总

### 关系类型

- **亲友** 👩‍👧 / 👨‍👦 / 👬 / 👭 / 👵：通过邀请链接加入或互相同意，默认互相关心，可看到全部允许分享的脱敏卡片。
- **关注的好友** ✨：通过"附近"或邀请码加入，需双方同意，仅看到陌生人级别的脱敏卡片。

### XP 经济

| 行为       | XP  | 上限    |
| -------- | --- | ----- |
| 点赞他人挑战   | +3  | 每日 60 |
| 给他人留言    | +5  | 每日 30 |
| 申请关注（附近） | +10 | 每日 50 |
| 成功添加好友   | +20 | 不设上限  |

### 隐私铁律

1. 任何人都看不到原始数值（步数、心率、深睡时长等具体数）。
2. 关心圈仅展示「挑战完成 / 进行中」状态 + 脱敏描述。
3. 用户可按挑战项分别控制"亲友 / 陌生人"两档可见性。
4. 药品 / 体重 / 健康状况强私密，默认关闭。
5. 位置信息精度最小 1 km，永不暴露精确位置。
6. 排行榜每周一凌晨重置今日 / 本周 XP，但累计 XP 永不重置。

### 通知规则

- 收到点赞 / 评论 / 好友请求 → 推送到锁屏（在系统通知中开启）。
- 通知频率：合并 5 分钟内的同人多次互动为 1 条。
- 用户可在 C6 组 5 关闭"接收亲友点赞通知"。

### 动态发送机制（v2.3 新增）

挑战分两类，发动态的时机不同：

#### 今日挑战（每日重复任务）

- **触发**：当日挑战**完成时**发送 1 条动态。
- **典型项**：走路 / 睡眠 / 喝水 / 跑步 / 站立 / 吃药 / 消耗。
- **理由**：今日挑战每天循环、无"开始时间"，且部分由戒指被动记录（如步数 / 心率），因此"完成时炫耀达成"是最自然的发送时机。未完成不发动态，避免噪声。

#### 练级挑战（长期主动加入）

- **触发**：分 5 个里程碑发送动态。

  | 触发时机 | `data-trigger` 值 | 文案语义 |
  | --- | --- | --- |
  | 加入挑战 | `joined` | "我加入了 X，期待 N 天后" |
  | 进度 25% | `progress-25` | "X 已完成 25% 里程碑" |
  | 进度 50% | `progress-50` | "X 已完成一半，节奏稳定" |
  | 进度 75% | `progress-75` | "X 已完成 75%，再坚持冲刺" |
  | 进度 100%（完赛） | `progress-100` | "X 完赛！等待抽奖" |

- **典型项**：30 天早睡早起 / 樱花漫步 30 天 50 公里 / 年度大挑战 等。
- **理由**：练级挑战是长周期 + 主动报名，需要在关键里程碑给出持续的社交反馈，维持热度并邀请好友互动。
- **去重**：同一挑战的同一里程碑只发 1 次（即使中途回退后再到达，也不重复发）。

#### 字段约定（C-Me 我的动态 / C5 好友主页 / C1.A 好友动态 共用）

每张动态卡 `.dyn-card`（或 `.my-dyn-card`）必须带：

- `data-feed-type="daily|level"`：动态类型。
- `data-trigger="completed|joined|progress-25|progress-50|progress-75|progress-100"`：触发原因。
- `data-time="ISO 时间戳"`：动态发送时间，列表按其倒序排（最新在最上）。
- 卡内副文行（`.dyn-time`）展示语义化时间 + 类型 + 触发标签，如：「今天 12 分钟前 · 今日挑战 · 完成」/「3 天前 · 练级挑战 · 加入」/「今天 09:42 · 练级挑战 · 50% 进度」。

#### 列表排序

C-Me 我的动态、C5 好友主页 TA 的动态、C1.A 好友动态（首页流）三处列表均按 `data-time` 倒序展示，无分页时一次性渲染。
