## 💡 用法小贴士

1. **首次对话先让 Claude 读规范**。每个新对话都是空白的，Claude 不会"记得"上次的规范。每次开新对话用模板时，Claude 会先 Read 你列出的 4 个文件，再开始动手 — 这是确保它遵循规范的关键。

2. **如果 Claude 跑偏了，引用规范行号纠正**。例如"按 00-设计规范.md §6.1 强约束，一级顶栏右侧只有 🌙 + 👤 两个按钮，请去掉你加的搜索按钮"。

3. **改完后让它自检**。在请求末尾加一句：

   > 完成后请检查：①是否所有颜色都用 CSS 变量；②是否所有 UI 图标都来自 SVG 库；③是否在 375px 宽度内验证过；④是否使用了规范定义的页面层级术语（中文）。

4. **每个模块文档命名规范**：
   - `01-今日.md` / `01-今日.html`
   - `02-关心.md` / `02-关心.html`
   - `03-个人中心.md` / `03-个人中心.html`
   - `04-练级.md` / `04-练级.html`
   - `05-收获.md` / `05-收获.html`

5. **跨模块引用别用类名**。规范要求用 ID（`T2` / `P6` / `C3.1`）+ 文档名（《03-个人中心.md》），不要写 `#today-page` 这种代码层引用。
