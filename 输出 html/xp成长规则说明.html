AIRING Growth System · XP / 等级 / 装备 / 特权

把 XP 做成一个长期成长系统，而不是只会跳数字的积分。

基于现有原型里的「今日挑战」「关心互动」「邀请好友」「累计 XP 换礼」这几条线，这套方案把 XP 明确分成今日反馈和长期成长两层。 用户每天通过健康行为与关心行为积累 XP，累计 XP 决定等级；等级再反向驱动收获页中的 IP 形象升级、装备变化和特权解锁。

核心闭环
🎯
完成今日挑战 走路、睡眠、消耗、喝水、站立、跑步、吃药，构成最稳定的 XP 来源。
❤️
发生关心互动 点赞、评论、邀请亲友加入，给中环和成长系统持续补能。
✨

🎁
解锁特权与权益 等级越高，可展示的名片、邀请权益、收获页内容与限定皮肤越完整。

👑
🦊

建议把特权具象化成 IP 装备位，用户一眼就能看懂自己“为什么更高级”。

1. 设计原则

这套 XP 规则不追求复杂，而是强调三个感受：每天都能涨、成长有目标、视觉奖励能被别人看见。

稳定

XP 主来源必须来自健康行为本身，不能让纯社交刷分压过真正的佩戴和打卡。这样才能保证戒指硬件价值始终是系统中心。

可炫耀

用户不是为了“数字”，而是为了“我看起来升级了”。所以每个等级都要对应清晰的外观变化和装备位增加。

可控

所有互动型 XP 都需要日上限、关系约束或有效行为判定，避免用户互刷、脚本刷、低质量评论刷分。

2. XP 分成两层

把“今日 XP”和“累计 XP”拆开，会让规则清晰很多。前者负责日反馈，后者负责长期成长。

Today XP
今日 XP Daily Feedback

今天通过挑战、点赞、评论、邀请等行为获得的 XP，用来驱动「成长圆环」和当天的反馈感。

用途：点亮成长圆环、触发今日 toast、展示“今日已获 +45 XP”。
节奏：每天清零，保证每天都有可追的短目标。
建议日目标：100 XP 作为成长圆环满格线，超过部分继续累计到长期 XP。
Total XP
累计 XP Long-term Growth

累计 XP 永不清零，直接决定等级、IP 形象、装备位和高级特权，是用户在收获页最想持续追的那条主线。

用途：升级、解锁新外观、提升邀请权益、进入更完整的收获页展示。
节奏：前期升级快，中后期拉长，避免太快毕业。
建议关系：所有今日 XP 同时计入累计 XP，不需要双记账心智。

3. XP 获取规则

以下数值尽量沿用你现有原型中的习惯：挑战奖励 20-40 XP，点赞 +3 XP，评论 +5 XP，邀请成功 +20 XP，再补上一些成长闭环奖励。

模块 行为 建议 XP 规则说明 限制
健康挑战 完成单个今日挑战 +20 ~ +40 沿用原型：喝水 / 吃药 20，站立 25，走路 / 跑步 30，消耗 35，睡眠 40。 同一挑战每天只结算 1 次。
健康挑战 今日等级徽章 铜 +10 / 银 +20 / 金 +30 和三环规则对齐。1 环 90% 解锁铜牌，2 环银牌，3 环金牌。 每天最高结算一次。
健康挑战 连续达标奖励 3 天 +10 / 7 天 +25 / 14 天 +50 奖励“坚持”，让用户愿意每天回来，而不是只刷一次高分。 按连续天数节点发放，不叠加重复领。
关心互动 点赞好友挑战 +3 与你现有关心页一致。点赞动作轻，适合做轻量补 XP。 每天最多计入 10 次，共 30 XP。
关心互动 发送一句关心 +5 保留现有评论奖励，但建议要求不少于 6 个字，且同一对象当天前 3 条有效。 每天最多计入 6 次，共 30 XP。
关心互动 回复点赞 / 完成互动闭环 +2 给“被关心后再回应”一点微奖励，强化关系黏性。 每天最多计入 5 次，共 10 XP。
关系成长 成功邀请好友加入 +20 与你现有文案一致。建议在“对方完成注册并产生首个互动”后结算，避免空注册。 每个新用户仅结算一次。
关系成长 亲友首次连接戒指 +50 这是硬件增长关键动作，奖励应该明显高于普通邀请。 每位亲友仅结算一次。
新手引导 首次连接戒指 / 完善档案 +20 / +10 用于把新用户快速送上 Lv2，尽早看到“升级”的乐趣。 一次性奖励。
建议将互动类 XP 的每日上限控制在 60-70 XP 之间，让健康挑战仍然占据当天主要 XP 贡献。

4. 等级阶梯

等级改成 10 级更合适。Lv1 是初始状态；从 Lv2 开始，每升一级只发生一件事：解锁或升级一件装备。这里最重要的不是等级名字，而是每一级到底给了用户什么具体特权。

披风 = 头像框 + XP 加倍 `Lv2 / Lv5 / Lv8` 解锁或升级披风。外显特权是头像框越来越华丽；奖励特权是 `XP × 1.05 / 1.15 / 1.25`。
头盔 = 头像皇冠 + App 皮肤 `Lv3 / Lv6 / Lv9` 解锁或升级头盔。外显特权是银 / 金 / 钻头像皇冠；奖励特权是解锁对应档位的 App 皮肤。
武器 = 头像等级标 + 实物奖励 `Lv4 / Lv7 / Lv10` 解锁或升级武器。外显特权是更强的头像等级标；奖励特权是进入不同档位的实物奖励池。
Lv1 初醒
0 - 99
初始等级，不给装备特权，先让用户建立“佩戴戒指 + 每天打卡”的基础习惯。
当前特权
无额外特权
仅保留基础头像与基础 IP 形象，作为成长起点。
Lv2 旅者
100 - 199
首次获得披风「微风披肩」。用户第一次明确感受到“升级会给我一件新装备”。
双特权
披风 I「微风披肩」: 基础头像框 + 所有有效 XP × 1.05
头像开始出现初阶披风框；从这一层开始，所有有效健康行为和互动行为都能吃到披风倍率。
Lv3 守望
200 - 349
获得第一顶头盔「银曜护额」。它的主要作用不是数值，而是让别人一眼认出你已经成长到更高等级。
双特权
头盔 I「银曜护额」: 银色头像皇冠 + 银阶 App 皮肤
在关心圈和练级页中，头像会出现银色皇冠装饰；同时解锁银阶 App 视觉皮肤。
Lv4 炽刃
350 - 549
获得第一把武器「夜蚀短刃」。武器不加 XP，直接给角色带来风格变化，让关心圈里的你更醒目。
双特权
武器 I「夜蚀短刃」: 青铜头像等级标 + 青铜实物奖励池
头像上出现首档等级标识；同时进入第一档实物奖励池，例如基础周边、贴纸、小配件。
Lv5 流光
550 - 849
披风升级为「星辉长披」。视觉上从轻薄披肩进化成更长、更亮、更有层次的主披风。
双特权
披风 II「星辉长披」: 进阶头像框 + 所有有效 XP × 1.15
头像框视觉更完整、更亮；比 Lv2 的披风倍率更高，长期价值进一步拉开差距。
Lv6 星冕

头盔升级为「鎏金战冠」。这一级开始，你在关心圈和练级中的头像会有明显身份感。

5. 装备系统与特权绑定

装备不是纯装饰，而是等级规则的载体。用户升到某一级时，看到的不只是“数值变了”，而是“我这件装备变强了，所以我拿到这个特权”。

🦊

披风：头像框 + XP 加倍

披风有两个直接特权。第一个是外显特权：头像框会随着披风升级变得更完整、更华丽；第二个是奖励特权：所有有效 XP 按披风档位获得倍率加成。

披风 I：基础头像框 + XP × 1.05 披风 II：进阶头像框 + XP × 1.15 披风 III：终级头像框 + XP × 1.25
⛑️
🦊

头盔：头像皇冠 + App 皮肤

头盔也有两个直接特权。第一个是外显特权：头像上会出现越来越高阶的皇冠装饰；第二个是奖励特权：解锁对应档位的 App 皮肤主题。

头盔 I：银色头像皇冠 + 银阶皮肤 头盔 II：金色头像皇冠 + 金阶皮肤 头盔 III：钻石头像皇冠 + 钻阶皮肤
⚔️
🦊

武器：头像等级标 + 实物奖励

武器同样有两个直接特权。第一个是外显特权：头像上的等级标识越来越强；第二个是奖励特权：进入更高一档的实物奖励池。

武器 I：青铜等级标 + 青铜奖励池 武器 II：白银等级标 + 白银奖励池 武器 III：钻石等级标 + 钻石奖励池

6. 特权映射建议

这里建议把三件装备统一成同一种理解方式。用户不需要背复杂系统，只需要记住：每件装备都有两个特权，一个负责“别人看见”，一个负责“我能拿到什么奖励”。

等级 → 装备 → 双特权

披风：外显特权是头像框，奖励特权是 XP 加倍。
头盔：外显特权是头像皇冠，奖励特权是 App 皮肤。
武器：外显特权是头像等级标，奖励特权是实物奖励。
设计原则：每升一级都必须能同时回答两个问题，“别人会看到我哪里变了”和“我实际多拿到了什么”。

两个展示场景

关心圈 / 练级：重点展示三件装备的外显特权，也就是头像框、头像皇冠、头像等级标。
收获 / 个人主页：重点展示三件装备完整穿戴后的 IP 成长结果。
奖励兑现：披风通过 XP 倍率兑现，头盔通过 App 皮肤兑现，武器通过实物奖励兑现。
核心关系：等级不是抽象数字，而是推动装备升级；装备升级后，再同时带来外显特权和奖励特权。

这一段专门回答“用户到底能看出哪些不一样”。建议至少挑 4 个关键档位做固定展示，让团队和用户都能一眼读懂等级成长。

头像差异：关心圈 / 练级中的展示

用户自己最在意这个

8. 防刷与风控规则

如果 XP 要和等级、外观、特权绑定，这部分一定要提前设好，不然后面会很难收。

互动上限

点赞与评论都设每日上限，互动只能做“补能”，不能变成主刷分渠道。

关系约束

只有已建立有效关系的对象，互动 XP 才能计入；陌生流量建议只给很少或不给 XP。

质量判定

评论建议要求最少字数、重复文案去重、短时间连续评论不重复结算。

邀请结算

邀请成功不是“填邀请码”即得分，而是对方完成注册并产生首个有效互动后才结算。

下面这个例子可以帮助团队快速理解这套规则在产品里会怎么跑起来。

🌙
完成睡眠挑战 +40 XP，健康行为成为当天主要贡献。
❤️
点赞 3 位亲友 3 × 3 XP = +9 XP，作为轻互动补能。
💬
发出 2 条有效关心消息 2 × 5 XP = +10 XP，强化关系反馈。
🥈
今日银牌徽章 2 个圆环达到 90%，额外 +20 XP。
结算结果
109 XP

今日 XP 满格，成长圆环点亮，同时累计 XP +109。

当天反馈：成长圆环满格，toast 连续弹出，收获感很即时。
长期成长：如果用户原本是 168 累计 XP，今天结束后会到 277，直接从 Lv2 升到 Lv3。
视觉变化：收获页 IP 会从“披风 I 微风披肩”升级到“新增头盔 I 银曜护额”，关心圈头像也会首次出现银色等级装饰。

10. 最终建议

如果只保留一句话，这套系统最核心的判断标准就是：

XP 不只是奖励行为，而是把“健康坚持”翻译成“角色成长”的语言。

对用户：我不是在打卡，我是在给自己的披风、头盔、武器升级。
对产品：10 级成长、装备强化、社交展示、个人收获，终于被同一套规则统一起来。
对后续设计：下一步最值得细化的是“Lv1-Lv10 的装备视觉稿”以及“关心圈头像装饰规范 + 个人主页全身 IP 展示规范”。
