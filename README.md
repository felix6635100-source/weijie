# 唯戒 / AIRing · 需求原型项目总览

> 这是一份**冷启动文档**。任何新对话只需读完本文（含本文链接到的核心文件）即可独立产出符合规范的需求文档与 HTML 原型，不需要回看历史对话。

## 1. 项目是什么

**唯戒（AIRing）** 是一款基于戒指可穿戴设备的 **健康追踪 + 社交化养成 App**。

- **核心理念**："你关心的人，和关心你的人。"
- **目标用户**：佩戴 AIRing 戒指、希望长期养成健康习惯并与亲友互相打气的人。
- **核心闭环**：
  - 今日（每日闭环）：戒指实时数据 + 7 张当日习惯卡。
  - 练级（长期闭环）：用户主动选择的主题挑战、KOL 课程、健康故事，跨周 / 月 / 年。
  - 关心（社交闭环）：好友互动 + 脱敏卡片。
  - 收获（成果闭环）：徽章 / 奖品 / 成就（待补）。

**当前阶段**：产品 + 设计阶段，本项目（需求原型）的主线工作是产出 **375px 移动端 HTML 原型**，不是工程化代码。工程化代码是容器 `唯戒/` 下两个**独立的兄弟工程**（`APP前端工程/`、`后台前端工程/`），各自有自己的仓库与 README，不在本项目主线工作范围。

## 4. 设计规范要点（必须遵守）

> 完整规则见 `doc/00-设计规范.md`。下面只列**最容易踩坑、必须刻进肌肉记忆的硬约束**。

### 4.1 Token 与样式

- 颜色 / 字号 / 间距 / 圆角 / 阴影一律走 CSS 变量，**禁止写死 hex / px / rgb**。
- HTML 原型必须 `<link rel="stylesheet" href="./唯戒-设计规范-tokens.css">`。
- 同一类信息在不同页面必须使用同一个语义 Token（如"已完成"恒为 `--color-semantic-success`）。
- 组件可以加修饰符变体（`.btn-primary` / `.is-success`），**禁止 fork 出"风格相近的新组件"**。
- z-index 用 token：`--z-sheet 200` / `--z-picker 400` / `--z-alert 600` / `--z-critical 800`。

### 4.2 视口与图标

- 容器宽度固定 **375px**（iPhone 标准），桌面查看居中显示。禁止"在桌面宽度下看着对就行"。
- UI 图标必须用 SVG Sprite：`<svg class="icon icon-md"><use href="#icon-xxx"/></svg>`，引用 `唯戒-icons-data.js`。
- 仅健康专用图标（HRV / 心率 / 跑步 / 戒指 / 药品等）允许使用 emoji，UI 图标禁用 emoji。
- 徽章系统专用 emoji：金 = 🏆 / 银 = 🥈 / 铜 = 🥉 / 铁 = 🪨 / 空 = ⚪（5 状态全局共识）。

### 4.3 全局结构（00-设计规范 § 6）

- 顶部由两层叠加：`.ios-status-bar`（44px）+ `.app-topbar--primary`（52px），共用同一片 **白色 70% 半透明 + 毛玻璃磨砂**（`rgba(255,255,255,0.7)` + `backdrop-filter: blur(20px) saturate(180%)`），形成连续"磨砂帽"。
- **一级页面顶栏右侧永远只有两个按钮：主题切换 + 个人设置。** 模块特定动作下沉到页面内部，禁止往顶栏加搜索 / 设置 / 关心设置等按钮。
- 一级 Tab 共 4 个，顺序：**今日 / 练级 / 关心 / 收获**。Tab Bar 在所有一级页面通用；进入二级页面 / 全屏页面 / 全屏弹层时隐藏。
- 标题在历史日下不变（一级页面标题恒等于模块名，日期上下文由页内的"一周日期切换条"承担）。

### 4.4 形式术语（00-设计规范 § 6.0）

业务文档"形式"列**只允许使用以下中文术语**，禁止英文（Bottom Sheet / Picker 等只能出现在代码 / 类名层）：

> 一级页面 · 二级页面 · 全屏页面 · 子步骤 · 内嵌选项卡 · 内嵌区段 / 内嵌卡片 · 浮层 · 底部弹层 · 滚轮选择器 · 操作菜单 · 确认框 · 轻提示 · 全屏弹层 · 全屏轻提示

### 4.5 隐私脱敏（00-设计规范 § 10）

- 步数 / 心率 / HRV / 睡眠分期 / SpO2 / 体温 — 永不对外暴露原始数值。
- 服药记录 / 剂量 / 药品名 — 永不外传。
- 精确位置 — 永不外传，仅可显示"约 N 公里"。
- 挑战完成状态 / 完成度百分比 — 默认可见亲友，可由用户开关调整范围。

## 5. 文档写作规范（00-设计规范 § 11）

每个页面 / 弹层必须按 **四段式** 写：

1. **状态（State）**：表格三列 `状态 / 触发条件 / 关键差异`。关键差异写"用户看到的不同"（如"按钮置灰"），禁止写代码层（如 `loading=true`）。
2. **元素构成（Element Composition）**：从上到下、从左到右罗列视觉元素。可以写元素类型 / 文案 / 视觉提示 / 复用的组件名，**禁止**写 HTML id / CSS class / CSS 变量 / 框架代码。
3. **操作逻辑（Interaction Logic）**：统一格式"用户行为 → 系统响应"，至少覆盖进入 / 退出 / 点击 / 输入 / 滑动 / 跳转 / 弹层。
4. **业务规则（Business Rules，可选）**：仅当存在非显而易见的业务约束时才写。

**跨模块引用**：必须用 ID（T2 / P6 / C3.1）+ 文档名（《03-个人中心.md》），不允许写代码层引用（如 `#today-page`）。

**索引表 5 列**：每个模块文档头部必须有 `ID / 名称 / 形式 / 触发方式 / 原型状态` 索引表；原型状态约定：⬜ 未开始 · 🚧 进行中 · ✅ 已完成。

## 7. HTML 原型必备的引入片段

每份新原型放在 `html/` 目录下，在 `<head>` / `<body>` 末尾需要这两段（相对路径，与 `tokens.css`、`icons-data.js` 同级）：

```html
<!-- 在 <head> 中 -->
<link rel="stylesheet" href="./唯戒-设计规范-tokens.css">

<!-- 在 </body> 前，注入 SVG Sprite -->
<script src="./唯戒-icons-data.js"></script>
<script>
  document.body.insertAdjacentHTML(
    'afterbegin',
    '<div style="position:absolute;width:0;height:0;overflow:hidden">'
    + window.WEIJIE_ICONS_SPRITE + '</div>'
  );
</script>
```

视口：

```html
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">
```

## 8. 提交前自检 Checklist

每次产出原型 / 文档前过一遍：

- 所有颜色 / 字号 / 间距是否都用了 `var(--xxx)`？没有写死 hex / px / rgb？

- 文档"形式"列是否只用了规范 § 6.0 的中文术语？

## 9. 克制原则（全项目通用）

> **只做被要求的事，不要自作主张添加额外内容。**
>
> - 让你去掉边框，就只去掉边框，不要顺手加阴影、加圆角、加背景色。
> - 让你加一个按钮，就只加按钮，不要顺手加 tooltip / 动画 / loading 状态。
> - 需求里没有提到的视觉效果、交互效果、样式属性，一律不加。
> - 如果你认为某个额外改动有必要，先问用户，不要直接做。

## 10. 文档与 Token 维护流程（00-设计规范 § 12）

1. 新增设计 Token 必须先在 `html/唯戒-设计规范-tokens.css` 的 `:root { ... }` 段加入。
2. 同步在 `doc/00-设计规范.md` 对应章节登记，并升级文档头部版本号 + 写明变更说明。
3. 同步在 `html/唯戒-设计规范预览-v1.8.html` 对应章节增加可视化示例。
4. 业务原型只允许复用 Token，不允许重写底层样式。
5. 颜色微调需验证亮 / 暗主题对比度（WCAG AA：正文 ≥ 4.5:1，大字号 ≥ 3:1）。
