唯戒 · 设计规范v1.8

本文档定义全局通用的设计 Token、组件、布局与交互规则，所有模块（今日 / 关心 / 练级 / 收获 / 个人中心）均必须遵循。新增样式或组件请先回到本文档确认，避免重复定义。

使用方式： 在所有模块 HTML 中引入 <link rel="stylesheet" href="./唯戒-设计规范-tokens.css">，仅使用本规范定义的 CSS 变量与组件类名，禁止写死颜色 / 像素 / 字号。
📱 预览基准：iPhone 375 × auto。 本规范中所有组件 / 布局 / 弹窗的预览效果都基于 iPhone 标准 375px 宽度容器渲染，与真机 App 视觉完全一致。后续业务模块原型也必须置于 375px 视口内开发，禁止在桌面宽度下"看着对就行"。Token / 文字 / 间距 / 圆角等纯文档展示不受此限制，可全宽呈现以便快速浏览。

设计原则

#原则说明
1一切走 Token颜色、字号、间距、圆角、阴影一律使用 CSS 变量，不允许在业务页面里写死 hex/像素。
2同源同色同一类信息在不同页面必须使用同一个语义 Token（如"已完成"恒为 --color-semantic-success）。
3iOS 体感优先弹窗、按钮顺序、滚动手势遵循 iOS 习惯（左取消 / 右确认）。
4隐私默认收紧涉及身体数据、药品、位置信息默认不外传，靠用户主动开启。
5可扩展不可 Fork组件可通过修饰符（.is-success / .btn-primary）变体，但禁止 fork 出"风格相近的新组件"。

9. 全局交互规则

#规则实现
1状态切换动画进入 / 退出 overlay 200ms ease-out，filter 不打断。
2数字增长百分比、XP 值入场用 0→目标的 800ms 缓动 + data-count 属性驱动。
3下拉刷新 / 滑动手势仅一级页面顶部允许下拉，二级页面禁用，防误触。
4错误反馈表单错误用红字 + 抖动 1 次（0.3s），不弹轻提示。
5加载态按钮内 24px spinner（白），按钮文字保留；列表用骨架屏。
6空状态emoji 图标 + 标题 + 引导句 + 主 CTA + 次 CTA。

10. 隐私 / 数据脱敏规范

数据类型默认对外可调
步数、心率、HRV、睡眠分期、SpO2、体温不外传不可开放原始数值
挑战完成状态（"已完成 / 进行中"）可见亲友用户开关：亲友 / 陌生人
挑战完成度百分比可见亲友用户开关
概括性时长区间（如"7~8h"）默认关用户开关
服药记录、剂量、药品名永不外传不可开放
精确位置永不外传仅可显示"约 N 公里"

12. 文档与 Token 维护流程

  1. 新增设计 Token 必须在 唯戒-设计规范-tokens.css 的 :root { ... } 段加入，并在本文档对应章节同步登记。
  2. 新增组件先在本预览页注册并验证；本文档对应章节追加描述与示例。
  3. 业务原型（如 今日+关心+登录-v2.html）只允许复用，不允许重写底层样式；如必须重写需走"设计评审"流程。
  4. 颜色微调需验证亮 / 暗主题对比度（WCAG AA：正文 ≥ 4.5:1，大字号 ≥ 3:1）。
业务文档写作模板（§ 11）： 所有模块需求文档（01 / 02 / 03 …）的每个页面 / 弹层 / 子模块必须按四段式描述：① 状态 / ② 元素构成 / ③ 操作逻辑 / ④ 业务规则（可选）。不写代码、不写实现细节，只写"用户看到 / 用户操作什么"。
